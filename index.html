知识点–概念：Oracle Database Release Numbers 新版本号规则

1. 介绍
    # 从 201707 开始
    # 从 18c 开始  
    # 之前为多年一发布
    # 之后为每年发布一个年度发行版 Release，之后追加季度更新  
    # 优点  
        - 能够将更多的新功能更快的提供给用户  
        - 大大减少了单次软件变更的数量  
        - 避免大的版本升级对系统产生的风险  
    # 网址
        - https://docs.oracle.com/en/database/oracle/oracle-database/18/upgrd/oracle-database-release-numbers.html#GUID-1E2F3945-C0EE-4EB2-A933-8D1862D8ECE2
2. 格式
    # <Major_Database_Release_version>.<Release_Update_version>.<Release_Updates_Revision_version>.<increment_version>.<reserved>
        - 对于大多数用户来说，只需要关注前 3 位  
    # <Major_Database_Release_version>
        - 表示发行年度  
        - 每年一发布  
    # <Release_Update_version>
        - 季度更新 RU 版本
        - 每年 1/4/7/10 月发布  
        - 主要包含查询优化器相关 BUG 和安全漏洞的修复等  
    # <Release_Updates_Revision_version>
        - 季度更新修订 RUR 版本
        - 对每一个 RU，概率性提供更新修订 RUR  
        - 对 RU 的缺陷进行的修复，并包含最新的安全漏洞补丁  
    # <increment_version>
        - Oracle 数据库的增量版本，偶尔会用于 Oracle 云数据库中  
    # <reserved>
        - 保留位，预留给将来  
3. 维护期  
    # 每个 Release 发布后的至少两年内，都会提供 RU 的支持  
    # 每个 RU 会提供两个独立的 RUR 支持  
    # 每个 RUR 之间间隔 6 个月  
    # 因此每个年度 Release 至少会有 3 年的支持周期

        # 数据量大

            - 数据分析  
            - 数据挖掘

0.2.1 sql 语法规则

知识点–概念：SQL 语法标准

    1.ORACLE sql  
        # WHERE 方式的连接  
    2.SQL1999  
        # JOIN 方式的连接  
        # SQL1999 compliant ANSI JOIN，SQL:1999 标准兼容  
    3.两种语法的性能相同

知识点–概念：SQL 语句规则

    1. SQL 语句不区分大小写  
    2. SQL 语句可以写在一行或多行，使用减号作为续行符  
    3. 关键字不可以缩写或跨行  
    4. 字符串不可以跨行  
    5. 使用 semicolon 表示语句结束  
    6. sqlplus 中  
        # 在交互界面，一次只能执行一条 sql 语句  
        # 在脚本中，一个脚本中可以包含多条 sql 语句

知识点–概念：sql 良好的编写习惯

    1. 使用缩进增加可读性  
    2. 子句通常单独一行  
    3. 关键字通常大写，其他内容通常小写，如表名或列名

0.2.2 sql statement

知识点–概念：sql statement 分类

    1. DML，select、insert、update、delete、merge  
    2. DDL，create、alter、drop、truncate  
    3. DCL，grant、revoke  
    4. TCL，commit、rollback、savepoint

知识点–概念：sql statement 结束

    1. sqlplus  
        # ;        
            - 常规的结束
            - 结束并加载到 sql buffer 中并运行  
        # /
            - 单独一行上只加一个 slash 符号
            - 结束并加载到 sql buffer 中并运行  
        # 空行
            - 一个空行
            - 结束并加载到 sql buffer 中，但不运行  
            - 空行的具体作用，还和 sqlplus 的 SQLBLANKLINES 参数有关

知识点–概念：sql keyword 关键字

    1. 字典  
        # V$RESERVED_WORDS        -- 记录数据库中的所有关键字

知识点–概念：sql 注释

    1. REM，REMARK  
    2. --   
    3. /*...*/  
    4. sql 注释最好放在单独一行中，并且不要嵌入在一个 sql statement 中

知识点–概念：sql 续行符

    1. sqlplus 命令如果太长一行写不下，则在行尾加一个减号作为续行符  
        # sqlplus 会自动删除所有的行尾减号，再将整个语句进行拼接  
        # sql statement 中的行尾减号也会被自动删除，如果可能导致语法错误，则将行尾减号移至第二行行首

知识点：Markdown 中需要转义的字符

    1.转义字符 \
        # \        反斜线
        # `        反引号
        # *        星号
        # _        下划线
        # {}    大括号
        # []    中括号
        # ()    小括号
        # #        井号
        # +        加号
        # -        减号（连字符）
        # .        句点
        # !        感叹号

知识点：markdown 块级元素

    1.空行
        # 不包含任何字符的行
        # 只包含空白字符的行
        # 空行被视为段落的间隔
        # 或连续的多个空行被解析为一个空行
    2.换行
        # 以两个或两个以上的空格结尾，再输入回车符
    3.水平线
        # 在一行里只放三个或更多个 -、*、_，就会得到一个水平线，一行中的字符必须相同
    4.atx 风格标题
        # # <title_level_1>
        # ## <title_level_2>
        # ### <title_level_3>
        # #### <title_level_4>
        # ##### <title_level_5>
        # ###### <title_level_6>
        # atx 风格标题也可以添加结尾 #，仅为了美观，结尾 # 的数量多少不影响标题的级别
    5.setext 风格标题
        # level_1 级标题，下一行仅包含 = 符号，一个即可
        # level_2 级标题，下一行仅包含 - 符号，至少一个即可，并创建水平线
            - 注意如果水平线是由 * 或 _ 字符创建，则不会将上一行识别为标题
            - 注意若 - 被识别为无序列表的开头，则不会将上一行识别为标题
        # setext 风格标题只支持到 level_2 级

知识点：行级元素

    1.字体格式
        # *<text>*            <-- 斜体
        # _<text>_            <-- 斜体
        # **<text>**        <-- 加粗
        # __<text>__        <-- 加粗
        # ~~<text>~~        <-- 删除线
        # <u><text></u>        <-- 下划线
        # 1.~~文本~~        <-- 删除线

知识点：markdown 引用块

    1. 引用块
        # 每一行在行首添加 >
        # 或仅在第一行的行首添加 >，其他行省略 > 
        # 添加多个 > 符号，进行嵌套饮用块
        # 引用块中的 Markdown 元素会被解析
        # 引用快遇到空行结束

知识点：markdown 代码和代码块

    1. 代码块
        # 每一行缩进四个 space 或一个 TAB
        # 代码块中的字符会直接打印，不解析
        # 代码块在遇到没有缩进的一行自动结束
    2. 行内代码
        # 使用一个或多个连续的反引号包住
        # 要在行内代码中添加反引号，则起始反引号和结束反引号使用多个连续的反引号，并使用空格间隔，则行内代码中数量较少的反引号会被解析为普通字符
    3. 代码高亮

知识点–概念：markdown 列表

    1. 无序列表
        # - <content>
        # + <content>
        # * <content>
        # 三种符号可以混用
    2. 有序列表
        # <N>. <content>
        # 有序列表的 <N> 可以随便写
            - 第一个 <N> 视为有序列表的开始序号
            - 之后的 <N> 会自动按照在有序列表中的位置进行进位
        # 有序列表可以嵌套有序列表，子层在父层的基础上，开头加两个 space
        # 有序列表可以嵌套无序列表，子层在父层的基础上，开头加四个 space
            - 子层的五序列表若开头仅加了两个 space，则与父层的有序列表平级
    3. 列表嵌套列表
        # 子层在父层的基础上，开头加两个 space 或一个 TAB
        # 无序列表可以嵌套无序列表或有序列表
        # 有序列表可以嵌套无序列表或有序列表

第12章 Reguler_Expression

知识点–概念：正则表达式规则

    1. 正则表达式区分大小写，case-sensitive

知识点–概念：基础元字符

    1. 点字符
        # .            ，除了换行符外的任意单个字符
    2. 字符集合
        # 字符集合中，除 -、^、[ ]、\ 以外，其他所有字符都恢复为普通字符
        # 字符集合，只能匹配单个字符，语法上也限定了其只能匹配单个字符
        # []        ，匹配方括号中包含的字符
        # [^]        ，匹配除了在方括号中的字符外的其他字符
        # [a-h]        ，指定字符范围，按照 ascii
        # [^a-h]    ，指定否定字符范围，按照 ascii
    3. 重复次数
        # *        ，匹配重复 0 个到无数个前字符
        # +        ，匹配重复 1 个到无数个前字符
        # ?        ，匹配重复 0 个或 1 个前字符
        # {n}    ，匹配重复 n 个前的字符
        # {n,}    ，匹配重复 n 个到无数前字符
        # {n,m}    ，匹配重复 n 个到 m 个前字符
    4. 连续特征字符串
        # (xyz)        ，匹配与 xyz 完全相等的字符串
        # (a|b|c)    ，匹配与 a 完全相同的字符，或与 b 完全相同的字符，或与 c 完全相同的字符
        # (a|b|xyz)    ，匹配与 a 完全相同的字符，或与 b 完全相同的字符，或与 xyz 完全相同的字符串
    5. 或运算
        # |        ，或运算符，两侧可接正则表达式，匹配符合前正则表达式所指定的字符，或匹配符合后正则表达式所指定的字符
    6. 锚点
        # ^        ，行首
        # $        ，行末
    7. 转义字符
        # \

知识点–概念：功能字符

    1. 字母和数字
        # \w    ，匹配所有字母数字加下划线，等同于 [a-zA-Z0-9_]
        # \W    ，匹配所有非字母数字和下划线，即除了下划线外的所有符号，等同于 [^\w]
        # \d    ，匹配数字，等同于 [0-9]
        # \D    ，匹配非数字，等同于 [^\d]
    2. 空格字符
        # \b    ，匹配一个单词边界，如 <space>、<period> 等
        # \B    ，匹配一个非单词边界
        # \f    ，匹配一个换页符
        # \r    ，匹配一个回车符 CR
        # \n    ，匹配一个换行符 LF
        # \t    ，匹配一个制表符
        # \v    ，匹配一个垂直制表符
        # \p    ，匹配 DOS 行终止符 CR/LF，等同于 \r\n
        # \s    ，匹配所有空格字符，等同于 [ \t\n\f\r\p]
        # \S    ，匹配所有非空格字符，等同于 [^\s]
    3. 向后引用 back reference
        # (<reguler_expression>)\<N>
            - 该子字符串与正则表达式的第 N 个用括号围起来的捕捉群 (capture group) 子表达式 (subexpression) 匹配
            - N 是从 1 开始的十进制正整数，其上限可能是 9、31、99 甚至无限
            - 范例
                '(.)\1' 表示匹配两个连续的相同字符

知识点–概念：POSIX 方括号表达式 POSIX Bracket Expressions

    1. 使用方法
        # grep '[[:digit:]]'
    2. [:alnum:]    ，代表英文字母和数字        ，亦即 A-Z、a-z、0-9
    3. [:alpha:]    ，代表英文字母                ，亦即 A-Z、a-z
    4. [:upper:]    ，代表英文大写字母            ，亦即 A-Z
    5. [:lower:]    ，代表英文小写字母            ，亦即 a-z
    6. [:digit:]    ，代表数字                    ，亦即 0-9
    7. [:xdigit:]    ，代表 16 进制会用到的字符    ，亦即 0-9、A-F、a-f
    8. [:blank:]    ，代表空白                    ，亦即 <space>、<tab>
    9. [:punct:]    ，代表标点符号                ，亦即 "、'、?、!、;、:、#、$ 等等
    10. [:print:]    ，代表任何可打印字符
    11. [:cntrl:]    ，代表控制键                ，亦即 CR、LF、<tab>、<del> 等等
    12. [:graph:]    ，代表空白以外其他所有按键
    13. [:space:]    ，代表任何会产生空白的字符    ，包括 <space>、<tab>、CR 等等

知识点–概念：零宽度断言 lookaround

    1. 正先行断言 positive lookahead
        # <search_reguler_expression>(?=<lookaround_reguler_expression>)
            - <search_reguler_expression>，搜索目标字符串，返回值为搜索目标字符串的匹配项
            - <lookaround_reguler_expression>，断言判断式
            - 要求目标字符串后面紧跟符合断言判断式的字符串
            - 范例
                'g(oo|ol)d(?=(ing|ed))'，搜索 good 或 gold，并且后面均紧接着 ed 或 ing
    2. 负先行断言 negative lookahead
        # <search_reguler_expression>(?!<lookaround_reguler_expression>)
        # 目标字符串后面紧跟不符合断言判断式的字符串
    3. 正后发断言 positive lookbehind
        # (?<=<lookaround_reguler_expression>)<search_reguler_expression>
        # 目标字符串前面为符合断言判断式的字符串
    4. 负后发断言 negative lookbehind
        # (?<!<lookaround_reguler_expression>)<search_reguler_expression> 
        # 目标字符串前面为不符合断言判断式的字符串

知识点–概念：模式修正 modifiers 或者 flag

    1. 忽略大小写
        # /.../i
    2. 全局搜索
        # /.../g，一次返回所有符合的结果
    3. 跨行搜索
        # /.../m
    4. 这三个 flag 是某种场景下的应用，不在正则的标准中，在 notepad 中都不适用

知识点–概念：贪婪匹配 Greedy 和懒惰匹配 lazy

    1. 贪婪匹配，尽可能匹配最长的字符串，默认
    2. 懒惰匹配，尽可能匹配最短的字符串，在重复次数符号后再加一个问号
        # *?
        # +?
        # {n,}?
