第05章 Markdown

知识点：markdown 介绍

    1.官网
        # https://daringfireball.net/projects/markdown/

知识点：Markdown 中需要转义的字符

    1.转义字符 \
        # \        反斜线
        # `        反引号
        # *        星号
        # _        下划线
        # {}    大括号
        # []    中括号
        # ()    小括号
        # #        井号
        # +        加号
        # -        减号（连字符）
        # .        句点
        # !        感叹号

知识点：markdown 块级元素

    1.空行
        # 不包含任何字符的行
        # 只包含空白字符的行
        # 空行被视为段落的间隔
        # 或连续的多个空行被解析为一个空行
    2.换行
        # 以两个或两个以上的空格结尾，再输入回车符
    3.水平线
        # 在一行里只放三个或更多个 -、*、_，就会得到一个水平线，一行中的字符必须相同
    4.atx 风格标题
        # # <title_level_1>
        # ## <title_level_2>
        # ### <title_level_3>
        # #### <title_level_4>
        # ##### <title_level_5>
        # ###### <title_level_6>
        # atx 风格标题也可以添加结尾 #，仅为了美观，结尾 # 的数量多少不影响标题的级别
    5.setext 风格标题
        # level_1 级标题，下一行仅包含 = 符号，一个即可
        # level_2 级标题，下一行仅包含 - 符号，至少一个即可，并创建水平线
            - 注意如果水平线是由 * 或 _ 字符创建，则不会将上一行识别为标题
            - 注意若 - 被识别为无序列表的开头，则不会将上一行识别为标题
        # setext 风格标题只支持到 level_2 级

知识点：行级元素

    1.字体格式
        # *<text>*            <-- 斜体
        # _<text>_            <-- 斜体
        # **<text>**        <-- 加粗
        # __<text>__        <-- 加粗
        # ~~<text>~~        <-- 删除线
        # <u><text></u>        <-- 下划线
        # 1.~~文本~~        <-- 删除线

知识点：markdown 引用块

    1. 引用块
        # 每一行在行首添加 >
        # 或仅在第一行的行首添加 >，其他行省略 > 
        # 添加多个 > 符号，进行嵌套饮用块
        # 引用块中的 Markdown 元素会被解析
        # 引用快遇到空行结束

知识点：markdown 代码和代码块

    1. 代码块
        # 每一行缩进四个 space 或一个 TAB
        # 代码块中的字符会直接打印，不解析
        # 代码块在遇到没有缩进的一行自动结束
    2. 行内代码
        # 使用一个或多个连续的反引号包住
        # 要在行内代码中添加反引号，则起始反引号和结束反引号使用多个连续的反引号，并使用空格间隔，则行内代码中数量较少的反引号会被解析为普通字符
    3. 代码高亮

知识点–概念：markdown 列表

    1. 无序列表
        # - <content>
        # + <content>
        # * <content>
        # 三种符号可以混用
    2. 有序列表
        # <N>. <content>
        # 有序列表的 <N> 可以随便写
            - 第一个 <N> 视为有序列表的开始序号
            - 之后的 <N> 会自动按照在有序列表中的位置进行进位
        # 有序列表可以嵌套有序列表，子层在父层的基础上，开头加两个 space
        # 有序列表可以嵌套无序列表，子层在父层的基础上，开头加四个 space
            - 子层的五序列表若开头仅加了两个 space，则与父层的有序列表平级
    3. 列表嵌套列表
        # 子层在父层的基础上，开头加两个 space 或一个 TAB
        # 无序列表可以嵌套无序列表或有序列表
        # 有序列表可以嵌套无序列表或有序列表
    4. 列表嵌套其他块
        # 列表嵌套代码块，代码块相对于父层，开头加两个 TAB
        # 列表嵌套饮用可，引用块相对于父层，开头加一个 TAB 和一个 >
    5. 列表的间隔
        # 标题可以间隔不同的列表块

知识点：markdown 表格

    1. 格式一
        # name | age
        # ---- | ---
        # LearnShare | 12
        # Mike |  32
    2. 格式二
        # |    name    | age |            <-- 指定列标题
        # | ---------- | --- |            <-- 指定列最小宽度和对齐方式
        # | LearnShare |  12 |
        # | Mike       |  32 |
    3. 表格内对齐
        # :---        <-- 左对齐
        # :--:        <-- 居中对齐
        # ---:        <-- 右对齐
    4. 表格内换行
        # <p> <内容> </p>
    5. 表格中的其他 Markdown 元素会被解析出来
        # 范例
            |     name     | age |             blog                |
            | ------------ | --- | ------------------------------- |
            | _LearnShare_ |  12 | [LearnShare](http://xianbai.me) |
            | __Mike__     |  32 | [Mike](http://mike.me)          |

知识点：markdown 工作列表

    1. 范例
        # 范例
            - [ ] Eat
            - [x] Code
              - [x] HTML
              - [x] CSS
              - [x] JavaScript
            - [ ] Sleep
            - [ ] \(Optional) Open a followup issue

知识点：markdown 链接和图片

    1. 行内链接
        # [<链接文本>](<链接URL> "<链接标题>")
        # 连接文本必须使用方括号包括起来
        # 链接 URL 可以使用尖括号包括起来
        # 连接标题可以省略
        # 连接标题可以放在下一行并使用额外的空格或TAB填充，这样较长的URL会比较美观
        # 范例
            [<Search>](https:www.google.com "Google_Search")
    2. 引用型链接
        # 定义链接
            链接的定义可以放在文章的任意位置
            解析时，连接的定义不会显示出来
            连接的
            [<链接标识符>]: <链接URL>  "<链接标题>"
            [<链接标识符>]: <链接URL>  '<链接标题>'
            [<链接标识符>]: <链接URL>  (<链接标题>)
        # 使用连接
            [连接标识符]
            [连接标识符] []
            [<连接文本>][<连接标识符>]
            [<连接文本>] [<连接标识符>]
        # 链接标识符可以由字母、数字、空格和标点符号组成，不区分大小写
    3. 自动链接

知识点：markdown 引用本地资源

    1. 格式
        # ./<path>/<file_name>
            . 表示 md 文件的位置
            <path> 表示本地资源相对于 md 文件的位置
            省略 ./<path>/ 表示资源位于 md 文件相同的目录







    3.自动连接
        # <URL>
    4.邮箱
        # <邮箱地址>
