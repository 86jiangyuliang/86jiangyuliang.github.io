知识点：markdown 介绍

知识点：Markdown 中需要转义的字符

    1.转义字符 \

知识点：markdown 块级元素

    1.空行
        # 不包含任何字符的行
        # 只包含空白字符的行
        # 空行被视为段落的间隔
        # 或连续的多个空行被解析为一个空行
    2.换行
        # 以两个或两个以上的空格结尾，再输入回车符
    3.水平线
        # 在一行里只放三个或更多个 -、*、_，就会得到一个水平线，一行中的字符必须相同
    4.atx 风格标题
        # # <title_level_1>
        # ## <title_level_2>
        # ### <title_level_3>
        # #### <title_level_4>
        # ##### <title_level_5>
        # ###### <title_level_6>
        # atx 风格标题也可以添加结尾 #，仅为了美观，结尾 # 的数量多少不影响标题的级别
    5.setext 风格标题
        # level_1 级标题，下一行仅包含 = 符号，一个即可
        # level_2 级标题，下一行仅包含 - 符号，至少一个即可，并创建水平线
            - 注意如果水平线是由 * 或 _ 字符创建，则不会将上一行识别为标题
            - 注意若 - 被识别为无序列表的开头，则不会将上一行识别为标题
        # setext 风格标题只支持到 level_2 级

知识点：行级元素

    1.字体格式
        # *<text>*            <-- 斜体
        # _<text>_            <-- 斜体
        # **<text>**        <-- 加粗
        # __<text>__        <-- 加粗
        # ~~<text>~~        <-- 删除线，注意 ~ 和 <text> 之间不能有空格
        # <u><text></u>        <-- 下划线

知识点：markdown 引用块

    1. 引用块
        # 每一行在行首添加 >
        # 或仅在第一行的行首添加 >，其他行省略 > 
        # 添加多个 > 符号，进行嵌套饮用块
        # 引用块中的 Markdown 元素会被解析
        # 引用快遇到空行结束

知识点：markdown 代码和代码块

    1. 代码块
        # 每一行缩进四个 space 或一个 TAB
        # 代码块中的字符会直接打印，不解析
        # 代码块在遇到没有缩进的一行自动结束
    2. 行内代码
        # 使用一个或多个连续的反引号包住
        # 要在行内代码中添加反引号，则起始反引号和结束反引号使用多个连续的反引号，并使用空格间隔，则行内代码中数量较少的反引号会被解析为普通字符
    3. 代码高亮

知识点：markdown 列表

    1. 无序列表
        # - <content>
        # + <content>
        # * <content>
        # 三种符号可以混用
    2. 有序列表
        # <N>. <content>
        # 有序列表的 <N> 可以随便写
            - 第一个 <N> 视为有序列表的开始序号
            - 之后的 <N> 会自动按照在有序列表中的位置进行进位
        # 有序列表可以嵌套有序列表，子层在父层的基础上，开头加两个 space
        # 有序列表可以嵌套无序列表，子层在父层的基础上，开头加四个 space
            - 子层的五序列表若开头仅加了两个 space，则与父层的有序列表平级
    3. 列表嵌套列表
        # 子层在父层的基础上，开头加两个 space 或一个 TAB
        # 无序列表可以嵌套无序列表或有序列表
        # 有序列表可以嵌套无序列表或有序列表
    4. 列表嵌套其他块
        # 列表嵌套代码块，代码块相对于父层，开头加两个 TAB
        # 列表嵌套饮用可，引用块相对于父层，开头加一个 TAB 和一个 >
    5. 列表的间隔
        # 标题可以间隔不同的列表块

第12章 Reguler_Expression

知识点：正则表达式介绍

    1. 网址
        # https://www.regular-expressions.info/reference.html
    2. 正则表达式区分大小写，case-sensitive

知识点：正则表达式基础字符

    1. 点
        # .            ，除了换行符外的任意单个字符
    2. 字符集合
        # 介绍
            - 字符集合中，除 -、^、[ ]、\ 以外，其他所有字符都恢复为普通字符
            - 字符集合中，只能匹配单个字符，语法上也限定了其只能匹配单个字符
        # []        ，字符集和，匹配方括号中包含的字符
        # [^]        ，字符集和补集，匹配除了在方括号中的字符外的其他字符
        # [a-h]        ，指定字符范围，按照 ascii
        # [^a-h]    ，指定否定字符范围，按照 ascii
    3. 重复次数
        # *        ，匹配重复 0 个到无数个前字符
        # +        ，匹配重复 1 个到无数个前字符
        # ?        ，匹配重复 0 个或 1 个前字符
        # {n}    ，匹配重复 n 个前的字符
        # {n,}    ，匹配重复 n 个到无数前字符
        # {n,m}    ，匹配重复 n 个到 m 个前字符
        # ?
            - 懒惰匹配 lazy，尽可能匹配最短的字符串，在重复次数符号后再加一个问号，如 *?、+?、{n,}?
            - 默认是贪婪匹配 Greedy，尽可能匹配最长的字符串
    4. 子字符串
        # (xyz)        ，匹配与 xyz 完全相等的字符串
        # (a|b|c)    ，匹配与 a 完全相同的字符，或与 b 完全相同的字符，或与 c 完全相同的字符
        # (a|b|xyz)    ，匹配与 a 完全相同的字符，或与 b 完全相同的字符，或与 xyz 完全相同的字符串
    5. 或运算
        # |        ，或运算符，两侧可接正则表达式，匹配符合前正则表达式所指定的字符，或匹配符合后正则表达式所指定的字符
    6. 锚点
        # ^        ，行首
        # $        ，行末
    7. 转义字符
        # \

知识点：正则表达式功能字符

    1. 字母和数字
        # \w    ，匹配所有字母数字加下划线，等同于 [a-zA-Z0-9_]
        # \W    ，匹配所有非字母数字和下划线，即除了下划线外的所有符号，等同于 [^\w]
        # \d    ，匹配数字，等同于 [0-9]
        # \D    ，匹配非数字，等同于 [^\d]

        # \r    ，匹配一个回车符 CR

        # \v    ，匹配一个垂直制表符

知识点：POSIX 方括号表达式 POSIX Bracket Expressions

    2. [:alnum:]    ，代表英文字母和数字        ，亦即 A-Z、a-z、0-9
    3. [:alpha:]    ，代表英文字母                ，亦即 A-Z、a-z
    4. [:upper:]    ，代表英文大写字母            ，亦即 A-Z

知识点：正则表达式捕获组 Capturing Groups 和回溯引用 backreference

    1. 介绍
        # 捕获组，写在查找字符串中
        # 回溯引用，写在替换字符串中
        # 通过使用捕获组和回溯引用，在进行字符串替换时，保留原始字符串中的一部分
            - 范例
                原文：aaa123bbb789ccc
                查找：aaa([1-3]*)bbb([7-9]*)ccc
                替换：xxx\1yyy\2zzz
                结果：xxx123yyy789zzz
        # 定义捕获组时，要注意捕获组前方的重复次数，如 * +，可能因为贪婪匹配而导致捕获组为空，可以改为懒惰匹配
            - 范例
                原文：aaa123bbb789ccc
                查找：.*([1-3]*)bbb([7-9]*)ccc
                替换：xxx\1yyy\2zzz
                结果：xxxyyy789zzz
                查找：.*?([1-3]*)bbb([7-9]*)ccc
                替换：xxx\2yyy\1zzz
                结果：xxx123yyy789zzz
    2. 捕获组格式
        # (<reguler_expression>)
            - 定义捕获组
            - 每个 (<reguler_expression>) 定义一个区域
    3. 回溯引用格式
        # \<N>
            - 定义回溯引用，N 为 1-9 这九个数，指向第 1-9 个捕获组
            - 指向未定义的捕获组时，结果为空值
            - 在 perl 和 python 中，N 可以指向第 1-99 个捕获组
        # ${N}
            - 定义回溯引用，N 为 1-99，指向第 1-99 个捕获组
            - 指向未定义的捕获组时，结果为空值

    3. 向后引用 back reference
        # (<reguler_expression>)\<N>
            - 该子字符串与正则表达式的第 N 个用括号围起来的捕捉群 (capture group) 子表达式 (subexpression) 匹配

知识点–概念：正则表达式断言 lookaround

    1. 断言介绍
        # 
    1. 正先行断言 positive lookahead
        # <search_reguler_expression>(?=<lookaround_reguler_expression>)
            - <search_reguler_expression>，搜索目标字符串，返回值为搜索目标字符串的匹配项
            - <lookaround_reguler_expression>，断言判断式
            - 要求目标字符串后面紧跟符合断言判断式的字符串
            - 范例
                'g(oo|ol)d(?=(ing|ed))'，搜索 good 或 gold，并且后面均紧接着 ed 或 ing
    2. 负先行断言 negative lookahead
        # <search_reguler_expression>(?!<lookaround_reguler_expression>)
        # 目标字符串后面紧跟不符合断言判断式的字符串
    3. 正后发断言 positive lookbehind
        # (?<=<lookaround_reguler_expression>)<search_reguler_expression>
        # 目标字符串前面为符合断言判断式的字符串
    4. 负后发断言 negative lookbehind
        # (?<!<lookaround_reguler_expression>)<search_reguler_expression> 
        # 目标字符串前面为不符合断言判断式的字符串
