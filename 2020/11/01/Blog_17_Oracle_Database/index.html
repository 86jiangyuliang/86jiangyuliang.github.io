知识点–概念：Oracle Database Release Numbers 新版本号规则

1. 介绍
    # 从 201707 开始
    # 从 18c 开始  
    # 之前为多年一发布
    # 之后为每年发布一个年度发行版 Release，之后追加季度更新  
    # 优点  
        - 能够将更多的新功能更快的提供给用户  
        - 大大减少了单次软件变更的数量  
        - 避免大的版本升级对系统产生的风险  
    # 网址
        - https://docs.oracle.com/en/database/oracle/oracle-database/18/upgrd/oracle-database-release-numbers.html#GUID-1E2F3945-C0EE-4EB2-A933-8D1862D8ECE2
2. 格式
    # <Major_Database_Release_version>.<Release_Update_version>.<Release_Updates_Revision_version>.<increment_version>.<reserved>
        - 对于大多数用户来说，只需要关注前 3 位  
    # <Major_Database_Release_version>
        - 表示发行年度  
        - 每年一发布  
    # <Release_Update_version>
        - 季度更新 RU 版本
        - 每年 1/4/7/10 月发布  
        - 主要包含查询优化器相关 BUG 和安全漏洞的修复等  
    # <Release_Updates_Revision_version>
        - 季度更新修订 RUR 版本
        - 对每一个 RU，概率性提供更新修订 RUR  
        - 对 RU 的缺陷进行的修复，并包含最新的安全漏洞补丁  
    # <increment_version>
        - Oracle 数据库的增量版本，偶尔会用于 Oracle 云数据库中  
    # <reserved>
        - 保留位，预留给将来  
3. 维护期  
    # 每个 Release 发布后的至少两年内，都会提供 RU 的支持  
    # 每个 RU 会提供两个独立的 RUR 支持  
    # 每个 RUR 之间间隔 6 个月  
    # 因此每个年度 Release 至少会有 3 年的支持周期

知识点–概念：sql 良好的编写习惯

    1. 使用缩进增加可读性  
    2. 子句通常单独一行  
    3. 关键字通常大写，其他内容通常小写，如表名或列名

1.1.6 GROUP BY 和 HAVING

知识点–语法：GROUP BY 基本语法

    1. 介绍  
        # GROUP BY 中不能使用聚合函数  
        # SELECT list 中的项，要么在 SELECT list 中以聚合函数的参数存在，要么写在 GROUP BY list 中作为分组依据  
    2. 语法  
        # GROUP BY {<column_name>|<function>|<expression>}[, ...]  
    3. 特性
        # 括号和先后顺序并不影响分组结果，只影响排序  
            - 以下三者结果相同  
                GROUP BY column_a, column_b, column_c

        # HAVING <having_condition>  
    2. HAVING 与 WHERE 的区别  
        # WHERE 是对 GROUP BY 前的虚拟表进行筛选，<where_condition> 中不可以使用聚合函数  
        # HAVING 是对 GROUP BY 之后的虚拟表进行筛选，<having_condition> 中可以使用聚合函数

            - DESC              <-- 降序  
            - NULLS FIRST       <-- 空值放在最前  
            - NULLS LAST        <-- 空值放在最后  
    2. <ordey_by_item>  
        # 可以是没有出现在 <select_list> 中的项  
            - <column_name>

1.1.8 OFFSET 和 FETCH

知识点–语法：OFFSET

    1. 功能  
        # 跳过前面 N 行  
        # 可以不和 FETCH 一起使用  
        # 处理的是 ORDER BY 之后的虚拟表
    2. 语法  
        # OFFSET <N> {ROW | ROWS}  
            - ROW、ROWS          <-- 语法提示词，无功能意义，但不能缺少

知识点–语法：FETCH

    1. 功能  
        # 只返回接下来的前 N 行  
        # 可以不和 OFFSET 一起使用  
        # 常用于 Top-N 查询
        # 处理 ORDER BY 之后再 FETCH 之后的虚拟表
    2. 语法  
        # [<offset_clause>] FETCH {FIRST|NEXT} {<N>| <percent> PERCENT} {ROW|ROWS} {ONLY | WITH TIES}  
            - FIRST、NEXT        <-- 语法提示词  
            - ROW、ROWS          <-- 语法提示词  
            - ONLY              <-- 指定了多少行，就返回多少行  
            - WITH TIES         <-- 必须配合 ORDER BY 子句使用，对排序的最后一个值将相同的行都取出，即使超出了 {<N>| <percent> PERCENT} 的限制

1.1.9 连接 JOIN

知识点–概念：连接

    1. 连接的意义  
        # 将数据分散储存，避免产生冗余数据  
    2. 等值连接、非等值连接  
        # 等值连接 equijoin  
            - 连接条件为等于的连接  
        # 非等值连接 nonequejoin  
            - 连接条件不是等于的连接  
            - 非等值连接虽然连接条件不是等于，但仍要尽量保证对每一行只有一个匹配行，否则容易产生大量冗余数据，一般没有实际意义  
    3. 内连接、外连接  
        # 内连接 inner join  
            - 内连接仅会保留符合条件的行  
        # 外连接 outer join  
            - 外连接会保留符合条件的行和不符合条件的行  
        # 左外连接 left outer join  
            - 保留左表的不符合条件的行  
        # 右外连接 right outer join  
            - 保留右表的不符合条件的行  
        # 全外连接 full outer join  
            - 左表和右表的不符合条件的行都会保留  
    4. 自连接 self join  
        # 左表和右表是同一张表  
        # 自连接必须使用表别名  
    5. 自然连接 natural join  
        # 自然连接是一种特殊的等值连接  
        # 自动寻找两张表中的公共列，自动进行等值连接  
        # 自然连接的前提  
            - table_A 和 table_B 有公共列  
            - 公共列的列名要相同  
            - 公共列的数据类型要相同，或者不同但可以进行隐式转换，否则会报错  
            - 如果有多个公共列，则所有的公共列都会被用作连接条件，并且必须满足每一对公共列的值都相等  
            - 多个公共列时，可能因为某一对公共列的问题而影响结果，或报错  
        # 自然连接的结果  
            - 会将公共列的重复项去重，并将公共列提到表的第一列  
            - 公共列中行的顺序按照左表中公共列的行顺序  
        # 对自然连接的结果进行查询时，如果引用公共列，则公共列不能加表别名  
    6. 交叉连接 cross join，笛卡尔乘积 Cartesian Products  
    7. 连接的方式  
        # sort_merge_join  
        # hash_join  
        # nest_loop

知识点–语法：USING，ON，指定连接条件

    1. 连接条件  
        # 作为连接条件的两个列，可以具有不同的数据类型，但必须可以相互进行隐式转换  
    2. USING  
        # 使用公共列自动进行等值连接，并会删除右表的公共列，将左表的公共列作为表的第一列  
        # 语法
            - USING (<column_name>[, ...])
    3. ON  
        # ON 方式会保留公共列，重名的公共列会重新起个名  
        # ON 方式是功能最全面的连接方式，能完成所有连接要求  
        # ON 方式最好使用表别名
        # 非等值连接只能使用 ON 方式  
        # 语法
            - ON (<join_condition> [{AND|OR} <join_condition>][ ...])

知识点–语法：连接 JOIN

    1. 自然连接 NATURAL JOIN  
        # FROM <table_a> NATURAL [INNER] JOIN <table_b>  
        # FROM <table_a> NATURAL {FULL|LEFT|RIGHT} [OUTER] JOIN <table_b>  
    2. 内连接 INNER JOIN  
        # FROM <table_a> [INNER] JOIN <table_b> USING (<column_name> [, ...])  
        # FROM <table_a> [INNER] JOIN <table_b> ON <condition>  
    3. 左外连接 LEFT OUTER JOIN  
        # FROM <table_a> LEFT [OUTER] JOIN <table_b> ON <condition>  
        # FROM <table_a> LEFT [OUTER] JOIN <table_b> USING (<column_name> [, ...])  
    4. 右外连接 RIGHT OUTER JOIN  
        # FROM <table_a> RIGHT [OUTER] JOIN <table_b> ON <condition>  
        # FROM <table_a> RIGHT [OUTER] JOIN <table_b> USING (<column_name> [, ...])  
    5. 全外连接 FULL OUTER JOIN  
        # FROM <table_a> FULL [OUTER] JOIN <table_b> ON <condition>  
        # FROM <table_a> FULL [OUTER] JOIN <table_b> USING (<column_name> [, ...])  
    6. 笛卡尔乘积  
        # FROM <table_a> , <table_b> ...  
        # FROM <table_a> CROSS JOIN <table_b> ...

知识点–概念：连接旧语法

    1. 等值连接，FROM table_A, table_B WHERE A_col = B_col  
    2. 左外连接，FROM table_A, table_B WHERE A_col = B_col(+)  
    3. 右外连接，FROM table_A, table_B WHERE A_col(+) = A_col  
    4. 笛卡尔乘积，FROM table_A, table_B, ...  
    5. 旧语法和新语法最好不要混用，有些情况下会报错

1.1.10 子查询

知识点–概念：子查询 subquery

    1. 介绍
        # 子查询，也称为内查询  
        # 父查询，也称为外查询、主查询  
        # 先执行子查询，后执行父查询  
    2. 子查询可以没有返回行  
    3. 子查询的嵌套和解嵌套  
        # 多数情况下，优化器会自动对子查询进行解嵌套，将子查询融合到主查询中  
        # 少数情况下，子查询不会被解嵌套，此时，对于主查询的每一个返回行，都会进行一次子查询，严重影响性能，包括以下情况
            * 层级查询  
            * 关联子查询  
            * 子查询中包含 ROWNUM  
            * 子查询包含集合运算符  
            * 子查询包含聚合函数  
    4. 内联视图 inline_view  
        # 也叫嵌入式视图  
            * 在 FROM 中使用子查询，将子查询的结果当做一个表来用  
            * 在 <select_list>、<condition> 上使用单行子查询，作为一个标量使用  
    5.单行子查询 single_row_subquery  
        # 返回值为一行一列的子查询  
        # 也称为标量子查询，因为 oralce 会将单行子查询当做一个标量进行处理  
        # 如果单行子查询没有返回行，则视为返回 NULL 值

知识点–概念：关联子查询 correlated subquery

    1. 介绍
        # 在子查询中，需要使用父查询的值作为 WHERE 条件，就称为关联子查询
        # 通常需要在父查询中定义表别名，在子查询中使用表别名  
        # 关联子查询，对主查询的每一行，都执行一次子查询  
            - 因而关联子查询的效率奇差
            - 建议使用连接进行替代  
                关联子查询次数多，占用空间少，属于时间换空间
                连接循环次数少，占用空间多，属于空间换时间
    2. 范例  
        # 查询每个部门的工资的前三名  
            - SELECT * FROM employees t_f WHERE employee_id IN  
                (SELECT employee_id FROM employees  
                    WHERE department_id = t_f.department_id AND salary IS NOT NULL  
                    ORDER BY salary DESC  
                    FETCH FIRST 3 ROWS WITH TIES )  
                ORDER BY department_id, salary DESC  
                ;

1.1.11 组合查询

知识点–概念：集合运算符，组合查询

    1. 集合运算符 Set Operators  
        # 实现数学上的集合运算  
            - UNION             <-- 并集  
            - UNION ALL         <-- 并集，不去重，不排序  
            - INTERSECT         <-- 交集  
            - MINUS             <-- 差集  
    2. 集合运算符的去重、保留 NULL、自动排序  
        # UNION ALL  
            - 不去重  
            - 不自动排序  
            - 保留 NULL  
        # UNION、INTERSECT、MINUS  
            - 去重  
            - 自动排序  
                以第一个查询的第一列为排序依据，升序排列  
            - 保留 NULL，对于多个 null ，合并为一个 null  
    3. 集合运算符和 ORDERY BY  
        # 每个查询不允许添加自己 ORDER BY  
        # 只能在总查询的最后添加总 ORDER BY  
    4. 集合运算符的优先级  
        # 集合运算符都具有相同的优先级  
        # 一个查询中出现多个集合运算符，按照从前向后的顺序，依次执行  
        # 括号可以改变集合运算符的优先级  
    5. 组合查询 Compound Query    
        # 使用集合运算符的查询称为组合查询  
        # 组合查询的 column  
            - 列以第一个查询为基础，不管其他的查询是否加了括号  
            - 所有其他查询的列的数量，要与第一个查询一致  
        # 组合查询的 column_name  
            - 列名以第一个查询为基础，不管其他的查询是否加了括号  
            - 其他所有查询的列名被忽略  
        # 组合查询的 data type  
            - 数据类型以第一个查询的数据类型的 data_type_group 为基础，不管其他的查询是否加了括号  
            - 其他所有查询的列的数据类型，要与第一个查询的对应的列，属于同一个 data_type_group  
            - 当数据类型为属于同一个 data_type_group 的不同数据类型，或宽度不同，则会进行数据类型的转换  
                如果两个查询都是 char 型，并且宽度相同，则结果为 char 型，具有相同宽度  
                如果两个查询都是 char 型，具有不同宽度，则结果为 varchar2 型，具有较宽的那个宽度  
                如果有任意一个为 varchar2 型，则结果为 varchar2 型  
            - 组合查询不会对数据类型进行跨 data_type_group 的数据类型的隐式转换  
    6. dummy column  
        # 使用 dummy column 来满足每个查询的数据类型  
        # dummy column 的值为 NULL  
        # 不需要为 NULL 指定数据类型，NULL 会自动进行类型转换

1.1.12 WITH 从句

知识点–语法：WITH 从句

    1. WITH 从句的作用  
        # 将子查询的结果储存在临时表空间中  
        # 在主查询中使用子查询的结果  
        # 提高效率，减少磁盘 IO  
    2. WITH 从句保存的是表，无法作为向量或标量使用  
        # 不能直接进行比较，需要在主查询中使用 select 进行取值  
        # 不能直接进行 IN 成员运算，需要在主查询中使用 select 进行取值  
    3. 语法  
        # WITH  
            <name_1> AS(<sub_query>) ,  
            <name_2> AS (<sub_query>) [, ...] ,  
        <main_query> ;

1.2 分组查询

1.2.1 结果分组

知识点–语法：GROUPING SETS，分组结果集

    1. 作用  
        # 将两个 GROUP BY 查询的结果，以 UNION ALL 的方式拼接在一起
        # 可以省略额外写一个子查询的烦劳
        # GROUP BY GROUPING SETS (<column_1>, <column_2>)  
            - 相当于 SELECT ... GROUP BY <column_1> UNION ALL SELECT ... GROUP BY <column_2>  
            - 相应的缺失的列，自动添加 NULL  
    2. 语法  
        # GROUP BY GROUPING SETS( <column_1>, <column_2> [, ...])  
        # GROUP BY GROUPING SETS( <column_1>, (<column_2>, <column_3>) [, ...])  
        # GROUP BY GROUPING SETS( ROLLUP( <column_1>, <column_2> ), CUBE( <column_3>, <column_4>) [, ...])  
    3. 范例  
        # GROUP BY GROUPING SETS ( ROLLUP(department_id, job_id), CUBE(department_id, job_id))  
            - 将以下的 GROUP BY 的结果，使用 UNION ALL 的方式拼接在一起  
            - GROUP BY ROLLUP(department_id, job_id)  
            - GROUP BY CUBE(department_id, job_id)  
        # GROUP BY GROUPING SETS ( ROLLUP(department_id, job_id), CUBE(first_name, last_name))  
            - 将以下的 GROUP BY 的结果，使用 UNION ALL 的方式拼接在一起  
            - 自动在相应的列上添加 NULL  
            - GROUP BY ROLLUP(department_id, job_id)  
            - GROUP BY CUBE(first_name, last_name)

知识点–语法：ROLLUP 滚动分组

    1. 介绍  
        # 从后向前，以滚动方式执行多个分组查询，将所有的分组结果拼接在一起  
        # 共计 n+1 个分组查询的结果，n 为分组项的个数  
        # 相应的列自动添加 NULL  
    2. 语法  
        # GROUP BY ROLLUP(<column_a>, <column_b> [, ...])  
        # GROUP BY GROUPING SETS( ROLLUP(<column_a>, <column_b>) [, ...])  
    3. 范例  
        # GROUP BY ROOLUP(column_a, column_b, column_c)  
            - 将以下的 GROUP BY 的结果拼接在一起  
            - GROUP BY column_a, column_b, column_c  
            - GROUP BY column_a, column_b  
            - GROUP BY column_a  
            - GROUP BY 0  
        # GROUP BY ROOLUP((column_a, column_b), column_c)  
            - 将以下的 GROUP BY 的结果拼接在一起  
            - GROUP BY column_a, column_b, column_c  
            - GROUP BY column_a, column_b  
            - GROUP BY 0

知识点–语法，CUBE 立方分组

    1. 以立方的方式执行多个分组查询，将所有的分组结果拼接在一起  
        # 展开所有的分组层次  
        # 共计 2^n 个分组查询的结果，n 为分组项的个数  
        # 相应的列自动添加 NULL  
        # ROLLUP 和 CUBE 的汇总行，在分组依据列自动为 NULL，所以在 WHERE 语句作用应该先过滤分组依据列为 NULL 的行  
    2. 用法  
        # GROUP BY CUBE(<column_a>, <column_b> [, ...])  
        # GROUP BY GROUPING SETS( CUBE(<column_a>, <column_b>) [, ...])  
    3. 范例  
        # GROUP BY CUBE(column_a, column_b, column_c)  
            - 将以下的 GROUP BY 的结果拼接在一起  
            - GROUP BY column_a, column_b, column_c  
            - GROUP BY column_a, column_b  
            - GROUP BY column_a, column_c  
            - GROUP BY column_b, column_c  
            - GROUP BY column_a  
            - GROUP BY column_b  
            - GROUP BY column_c  
            - GROUP BY 0

1.2.2 分组函数

知识点–函数：GROUP_ID

    1. 介绍
        # 当分组产生冲突集时，用来区分冲突集  
    2. 格式
        # group_id()        -- 空函数  
    3. 范例  
        # SELECT department_id, job_id, count(*), group_id()  
            FROM employees  
            GROUP BY GROUPING SETS( ROLLUP(department_id, job_id), CUBE( department_id, job_id))  
            ;

知识点–函数：GROUPING

    1. 介绍
        # 查看此行是否是 ROLLUP 或 CUBE 的汇总行  
        # GROUPING 只能接受一个参数  
    2. 格式 
        # GROUPING(<column_name>)  
            * 1 表示是汇总行
            * 0 表示不是汇总行  
    3. 范例  
        # SELECT department_id,job_id, count(*), grouping(department_id), grouping(job_id)  
            FROM employees  
            GROUP BY ROLLUP( department_id, job_id)

知识点–函数：GROUPING_ID

    1. 介绍
        # 查看此行是否是 ROLLUP 或 CUBE 的汇总行
        # 功能比 GROUPING 更多一些  
        # 一个参数时，则功能与 GROUPING 相同  
        # 多个参数时，意义将每一列按照 GROUPING() 得到结果，将所有结果按顺序拼接在一起，成为一个二进制数，返回其十进制的值

    2. 格式  
        # GROUPING_ID(<column_name>)  
        # GROUPING_ID(<column_1>, <column_2> [, ...])  
    3. 范例
        # GROUPING(department_id, job_id)
            - 某一行的返回值 2，则二进制数为 10，表示是 department_id 的汇总行，但不是 job_id 的汇总行

1.3 层级查询

1.3.1 层级查询

知识点–概念：层级查询 hierarchical query

    1. 介绍  
        # 查询数据中所包含的树形结构逻辑  
        # 结果是一个倒置树 inverted tree  
    2. 四种节点  
        # 根节点 root  
        # 父节点 parent  
        # 子节点 child  
        # 叶节点 leaf  
    3. 层级查询，需要表中有一列作为记录树形结构的逻辑关系，PRIOR 运算符  
        # 必须保证每一个节点都是唯一的，不能出现重复节点，否则会出现重复树  
        # 必须保证每一个子节点只有一个父节点，否则报错  
        # 例如 hr.employees 中的 manager_id 列  
        # 使用 PRIOR 运算符在 connect_by_clayse 中，来指定父子关系  
    4. 层级查询子句，放在 where clause 和 group clause 之间  
    5. 层级查询执行，在 join 之后执行，在 where clause 之前执行  
    6. 不建议在层级查询中使用 order by 和 group by，因为它们会冲掉最终的层级结果，不过可以使用 ORDER SIBLINGS BY

1.3.2 层级查询语法

知识点–语法：CONNECT BY 层级查询

    1. 语法  
        # SELECT ... FROM ... WHERE ... CONNECT BY [NOCYCLE] <connect_by_condition> [START WITH <start_condition>] ORDER BY ... ;  
        # SELECT ... FROM ... WHERE ... START WITH <start_condition> CONNECT BY [NOCYCLE] <connect_by_condition> ORCER BY ... ;  
            - NOCYCLE  
                显示环回的行，而不是报错  
                环回表示，一个节点，同时是另一个节点的祖先和后代  
            - CONNECT BY <connect_by_condition>  
                指定父子关系，确定树的逻辑关系  
            - START WITH <start_condition>  
                指定根节点，用来限定是查询一整棵树还是一棵分支  
                如果省略，则会返回所有的树，亦即所有子树都会被当成一个单独的树重复写一遍  
    2. <start_condition>  
        # 用来指定根节点的 child_item 项的性质，例如 employee_id = 100  
    3. <connect_by_condition>  
        # PRIOR 运算符  
            - 一元运算符，只能用在层级查询的 <connect_by_condition> 中，用来指定父节点项，以确定父子关系  
            - <connect_by_condition> 中必须有 PRIOR  
            - <connect_by_condition> 可以包含多个条件，但只能有一个条件需要指定 PRIOR，although you can have multiple PRIOR conditions  
            - PRIOR 放在等号左边或右边都可以  
            - PRIOR 一般都是指定的是 =，也可以使用其他运算符如比较运算符，但容易产生环回  
            - 格式  
                PRIOR <parent_item> = <child_item>  
                <child_item> = PRIOR <parent_item>  
            - 范例  
                PRIOR employee_id = manager_id          -- 子节点的 manager_id 指向父节点的 employee_id  
    4. <connect_by_condition> 和 <where_condition>  
        # <connect_by_condition> 先于 <where_condition>  
        # <connect_by_condition> 是在树的产生之前进行筛选，不满足条件的节点，连同其子孙，整个树枝都会被减掉  
        # <where_condition> 是在树的产生之后进行筛选，不满足条件的节点被删除，但子孙不受影响，因而会产生不连续的树  
    5. 层级查询的执行过程  
        # 通过 START WITH <start_condition>，确定根节点  
        # 找到根节点的子节点  
        # 递归找寻每个子节点的子节点，最终确立整个倒置树

知识点–语法：ORDER SIBLINGS BY 层级查询排序

    1. 介绍  
        # 层级查询排序，只能出现在层级查询中  
        # 按父节点进行分组  
        # 只对属于同一个父节点的所有子节点进行排序

1.3.3 层级查询相关元素

知识点–概念：CONNECT_BY_ROOT 运算符，返回根节点的某列的值

    1. 介绍  
        # 单目运算符，只能用在层级查询中  
        # 如果层级查询没有指定根节点，导致层级查询中有多棵树，则可以用 CONNECT_BY_ROOT 可以用来区分不同的树  
        # 可以放在 <select_list> 或 WHERE 中，不能用在 START WITH 和 CONNECT BY 中  
    2. 用法  
        # SELECT CONNECT_BY_ROOT <column_name> FROM <talbe_name> <hierarchical_query_clause> ;  
        # SELECT <expr_1>, <expr_2> FROM <table_name> WHERE CONNECT_BY_ROOT <column_name> = '321' <hierarchical_query_clause> ;

知识点–函数：SYS_CONNECT_BY_PATH，在层级查询中创建到根节点的路径

    1. 介绍  
        # 只能用在层级查询中  
        # 对每一个节点，创建其到根节点的路径  
    2. 语法  
        # SYS_CONNECT_BY_PATH ( <column_name>, <sep_char> )  
            - <column_name>     -- 放在路径中的值  
            - <sep_char>        -- 指定分隔符

知识点–概念：CONNECT_BY_ISCYCLE，层级查询伪列，检查是否出现环回

    1. 介绍  
        # 必须配合 NOCYCLE 一起使用  
        # 检查是否出现环回  
            - 1 表示出现了环回，即自己就是自己的父节点或祖先节点 ancestor  
            - 0 表示没有环回

知识点–概念：CONNECT_BY_ISLEAF，层级查询伪列，检查节点是否是叶节点

    1. 介绍  
        # 检查节点是否是叶节点  
            - 1 表示是叶节点  
            - 0 表示不是叶节点

知识点–概念：LEVEL，层级查询伪列，查询节点的深度

    1. 介绍  
        # 查看节点在倒置树中位于底基层  
            - 1 表示根节点  
            - N 表示根节点的子节点

知识点–操作：层级查询结合 case_expression 生成格式化树形结构

1.4.1 时间有效性查询

知识点–概念：时间有效性查询 temporal validity

    1. 时间有效性  
        # 在表中储存数据的有效时间段  
        # 查询时，需要指明有效时间段  
        # 仅返回符合有效时间段的行  
        # ILM information lifecycle mamagement  
    2. 时间有效性字段 valid_time_column  
        # 自动设置为隐藏列形式  
        # 用来作为进行时间有效性判断的依据，储存有效时间的时间段  
        # 实际会作为两个列进行储存数据，<valid_time_column_name>_START，<valid_time_column_name>_END，标识有效时间的时间段  
            - 可以显示指明 START 列和 END 列  
            - 也可以仅指明 valid_time_column_name，由 oracle 隐式自动转换为 START 列和 END 列  
            - CREATE TABLE 语句中，使用 PERIOD FOR 从句指明时间有效性字段  
    3. 时间有效性查询的两种方式  
        # 基于时间点的查询  
            - 查询时给出时间点  
            - 返回给定时间点位于有效时间段内的行  
            - SELECT 语句中，使用 AS OF PERIOD FOR 实现  
        # 基于时间段的查询  
            - 查询时给出时间段  
            - 返回给定时间段与有效时间段有重合的行  
            - SELECT 语句中，使用 VERSIONS BETWEED PERIOD FOR 实现  
    4. DBMS_FLASHBACK_ARCHIVE.ENABLE_AT_VALID_TIME  
        # 在 session 级指定有效时间点  
        # 之后所有的查询和 DML 自动进行对有效时间的基于时间点的查询  
        # 注意参数 ASOF 和 CURRENT 要大写，否则过程不会生效，并且不会给出错误提示

1.4.2 时间有效性查询语法

知识点–语法：时间有效性查询语法

    1. CREATE TABLE 指定时间有效性字段  
        # CREATE TABLE <table_name> ( ..., PERIOD FOR <valid_time_column_name> ) ;  -- 隐式方式  
        # CREATE TABLE <table_name> ( ...,                                          -- 显示方式  
            <valid_time_column_name>_START DATE,  
            <valid_time_column_name>_END DATE,  
            PERIOD FOR <valid_time_column_name> (<valid_time_column_name>_START, <valid_time_column_name>_END))  
            ;  
    2. ALTER TABLE 添加删除时间有效性字段  
        # ALTER TABLE <table_name> ADD PERIOD FOR <valid_time_column_name> [(<start_time_column>, <end_time_column>)] ;  
        # ALTER TABLE <table_name> DROP PERIOD FOR <valid_time_column_name> ;  
    2. INSERT 插入数据到表  
        # INSERT INTO <table_name> (..., <column_name>_START , <column_name>_END ) VALUES (...) ;  
    3. SELECT 进行时间有效性查询  
        # SELECT ... FROM ...       -- 基于时间点的查询  
            AS OF PERIOD FOR <valid_time_column> <datetime>  
            WHERE ...  
            ;                                                     
        # SELECT ... FROM ...       -- 基于时间段的查询  
            VERSIONS PERIOD FOR <valid_time_column> BETWEEN {<datetime>|MINVALUM} AND {<datetime>|MAXVALUM}  
            WHERE ...  
            ;

            - 已初始化的元素占据空间  
            - 未初始化的位置不占空间，不可使用，否则报错  
            - 可以进行跳跃赋值，其内部始终进行排序  
        # 只包含两列  
            - 主键列  
            - 数据列

知识点–语法：使用 ASSOCIATIVE ARRAYS 关联数组

    1. 定义 ASSOCIATIVE ARRAYS 类型  
        # TYPE <type_name> IS TABLE OF <data_type> [NOT NULL] INDEX BY {PLS_INTEGER|BINARY_INTEGER|VARCHAR2(<n>)} ;  
            - <data_type>  
                <data_type>  
                <table.column>%TYPE  
                <var_name>%TYPE                                         <-- 通过变量匹配数据类型  
                <table_name>%ROWTYPE  
            - INDEX BY {PLS_INTEGER|BINARY_INTEGER|VARCHAR2(<n>)}       <-- 指定关联数组的索引类型  
                INDEX BY VARCHAR2                                           <-- index 是字符串  
                INDEX BY PLS_INTEGER                                        <-- index 为整数，顺序为升序小到大，溢出了会报错，性能较好  
                INDEX BY BINARY_INTEGER                                     <-- index 为整数，顺序为升序小到大，溢出了不会报错，性能差些  
    2.声明 ASSOCIATIVE ARRAYS 变量

    3.使用 ASSOCIATIVE ARRAYS 变量  
        # <var_name>(<index>) := <value> ;  
        # <var_name>(<index>).<column_name> := <value> ;                <-- %ROWTYPE 型的关联数组，使用时还需要指定列名

知识点–概念：PLSQL 内嵌事务控制语句，PLSQL 的 ACID

    1. PLSQL 中可以内嵌 COMMIT，ROLLBACK，SAVEPOINT  
        # 事务控制为线性方式  
            - 子块中进行 COMMIT，会将父块中的未提交事务一起提交  
            - 最好将事务控制语句放在父块中  
            - 最好将事务控制语句放在每一个对子块的调用之后  
    2. PLSQL 支持语句级的 ACID，而不是整个 PLSQL 块级别的 ACID  
        # 如果 PLSQL 块中的某一条 SQL 语句存在错误，可能出现 PLSQL 块中前面几条 SQL 语句已经执行成功，但后面的 SQL 语句没有执行的情况

21.3.2 PLSQL 块嵌套

    1. BEGIN END 之间可以嵌套其他 PLSQL 块  
    2. EXCEPTION 中可以嵌套其他 PLSQL 块  
    3. PLSQL 变量在嵌套块中的作用范围  
        # 父块中的 PLSQL 变量可以传递给子块  
        # 子块中的 PLSQL 变量无法传递给父块  
        # 子块中的 PLSQL 变量如果与父块中的 PLSQL 变量重名，则在子块中屏蔽父块的 PLSQL 变量，当子块运行结束时，恢复父块的 PLSQL 变量  
        # 子块中，如果对父块的 PLSQL 变量进行了赋值，则只会在子块内生效，子块结束后变量会恢复在父块中的值  
    4. 绑定变量在嵌套块中的作用范围  
        # 绑定变量不受父块子块的影响  
        # 绑定变量的值取决于运行先后，先进行的修改会传递到之后的运行中

21.4 游标 cursor

21.4.1 游标介绍

知识点–概念：游标 Cursor，隐式游标 Implicit Cursor

    1. 游标  
        # 是一个指向 PGA 的某块区域的指针，与会话所绑定  
        # 用来处理 SELECT 语句的结果  
    2. 隐式游标  
        # SELECT 语句自动使用游标，称为隐式游标  
    3. 隐式游标的三个属性 attribute  
        # SQL%FOUND         <-- 布尔值，如果最近一次 SQL 语句有返回行则为 ture  
        # SQL%NOTFOUND      <-- 布尔值，如果最近一次 SQL 语句没有返回行则为 true  
        # SQL%ROWCOUNT      <-- 整型，最近一次 SQL 语句返回的行的数量  
    4. 字典  
        # V$OPEN_CURSOR  
    5. 范例  
        # SQL%NOTFOUND 范例
            -   BEGIN  
                    DELETE FROM employees WHERE salary < 5000 ;  
                    IF SQL%NOTFOUND THEN  
                        DBMS_OUTPUT.PUT_LINE(' No salary lower than 5000 .') ;  
                    ELSE  
                        DBMS_OUTPUT.PUT_LINE( SQL%COUNT || ' rows deleted .') ;  
                    END IF ;  
                END ;  
                /  
        # SQL%ROWCOUNT 范例
            -   BEGIN  
                    DELETE FROM employees WHERE salary < 5000 ;  
                    DBMS_OUTPUT.PUT_LINE( SQL%ROWCOUNT||' rows deleted') ;  
                END ;  
                /

21.4.2 显式游标

知识点–概念：显式游标 Explicit Cursor

    1. 显式游标的使用流程  
        # DECLARE 中，使用 CURSOR 定义游标  
        # BEGIN 中，使用 OPEN 打开游标 
        # BEGIN 中，使用 FETCH 取数据  
        # BEGIN 中，使用 CLOSE 关闭游标  
    2. 显示游标的用处
        # FOR 循环可以自动打开游标关闭游标  
        # 显式游标可以作为 %ROWTYPE 的基准，用于确定变量的类型  
            - <var_name> <cursor_name>%ROWTYPE ;  
        # 显式游标推荐用在 FOR 循环和 LOOP 循环中，不推荐用在 WHILE 循环中  
    3. 显式游标的属性  
        # %ISOPEN           <-- 布尔值 
        # %ISFOUND          <-- 布尔值，最近一次 FETCH 行为是否找到行  
        # %ISNOTFOUND       <-- 布尔值，最近一次 FETCH 行为是否没有找到行  
        # %ROWCOUNT         <-- 整型，游标中已经使用的行的数量

知识点–语法：CURSOR 声明显式游标

    1. 语法  
        # CURSOR <cursor_name> IS <select_statement> ;  
    2. 范例  
        # 声明显式游标  
            - DECLARE  
                CURSOR v_c01 IS select * from employees ;  
                CURSOR v_c02 IS select * from departments where department_id < 100 ;  
        # 定义显式游标并为关联数组声明类型  
            - DELCARE  
                CUROSR v_c01 IS select * from employees ;  
                TYPE v_t IS TABLE OF v_c01%ROWTYPE INDEX BY PLS_INTEGER ;  
                v_t01 v_t ;

知识点–语法：OPEN CLOSE 打开和关闭显式游标

    1. 语法

        # CLOSE <cursor_name> ;

知识点–语法：FETCH INTO 从显式游标中取出数据

    1. 介绍  
        # FETCH 从游标中取数据，必须整行取  
        # 当游标中的数据被取光，FETCH 不会对变量置空，变量会保存上一次 FETCH 到的数据  
        # 在 LOOP 循环中，最好在 FETCH 语句后直接跟 EXIT <cursor_name>%NOTFOUND 语句，否则最后一次循环是重复值  
    2. 语法  
        # FETCH <cursor_name> INTO ... ;

知识点–操作：使用显示游标

    1. 操作，打印 employees 所有的员工的 first_name 和 salary  
        #   DECLARE  
                v_sal NUMBER ;  
                v_name VARCHAR2(20) ;  
                CURSOR v_emp_cursor IS SELECT first_name, salary FROM employees ;  
            BEGIN  
                OPEN v_emp_cursor ;  
                LOOP  
                    FETCH v_emp_cursor INTO v_name, v_sal ;

                v_1 t_1 ;  
                v_count NUMBER := 1 ;  
            BEGIN  
                OPEN c_sal_cursor ;  
                LOOP  
                    FETCH c_sal_cursor INTO v_1(v_count) ;

                DBMS_OUTPUT.PUT_LINE( v_1(v_count).employee_id ||' '|| v_1(v_count).first_name || ' '|| v_1(v_count).salary) ;

            BEGIN  
                OPEN v_emp_cursor ;  
                LOOP  
                    FETCH v_emp_cursor INTO v_emp ;  
                    EXIT WHEN v_emp_cursor%NOTFOUND ;

21.4.3 显式游标其他使用方式

知识点–概念：带参数的显式游标

    1. 介绍  
        # 在声明游标时，确定游标的结构，同时为游标指明一个形参，不直接写死游标的内容  
            - 需要指定参数的数据类型
            - 在 SELECT 语句中需要将形参放在 WHERE 语句中来确定筛选条件
        # 在打开游标时，通过给定实参，确定游标真正包含的行  
        # 通常将游标的参数放在   
    2. 语法  
        # DECLARE  
            CURSOR <cursor_name> ( <paramenter_1_name> <data_type>, <parameter_2_name> <data_type> ...) IS <select_statement> ;  
         BEGIN  
            OPEN <cursor_name>( <parameter_1_value>, <parameter_2_val> ...) ;  
            <statements> ;  
            CLOSE <cursor_name> ;  
    3. 范例  
        # 取出工资超过 10000 的员工的 employee_id，OPEN CLOSE 方式  
            -   DECLARE  
                    CURSOR v_emp_cursor (a NUMBER) IS SELECT * FROM employees WHERE salary > a ;  
                    v_emp v_emp_cursor%ROWTYPE ;  
                BEGIN  
                    OPEN v_emp_cursor(10000) ;  
                    LOOP  
                        FETCH v_emp_cursor INTO v_emp ;  
                        EXIT WHEN v_emp_cursor%NOTFOUND ;  
                        DBMS_OUTPUT.PUT_LINE( v_emp.employee_id) ;  
                    END LOOP ;  
                    CLOSE v_emp_cursor ;  
                END;  
                /  
        #  取出工资在 3000 到 5000 的员工，和工资在 10000 到 15000 的员工，其中游标被 OPEN CLOSE 了两次  
            -   DECLARE  
                    CURSOR v_emp_cursor (a NUMBER, b NUMBER) IS SELECT * FROM employees WHERE salary BETWEEN a AND b order by salary ;  
                    v_emp v_emp_cursor%ROWTYPE ;  
                BEGIN  
                    DBMS_OUTPUT.PUT_LINE( '===='||' 3000 to 5000 '||'====') ;  
                    OPEN v_emp_cursor(3000, 5000) ;  
                    LOOP  
                        FETCH v_emp_cursor INTO v_emp ;  
                        EXIT WHEN v_emp_cursor%NOTFOUND ;  
                        DBMS_OUTPUT.PUT_LINE( v_emp.employee_id ||' '|| v_emp.salary) ;  
                    END LOOP ;  
                    DBMS_OUTPUT.PUT_LINE( '===='||' 10000 to 105000 '||'====') ;  
                    CLOSE v_emp_cursor ;  
                    OPEN v_emp_cursor(10000, 15000) ;  
                    LOOP  
                        FETCH v_emp_cursor INTO v_emp ;  
                        EXIT WHEN v_emp_cursor%NOTFOUND ;  
                        DBMS_OUTPUT.PUT_LINE( v_emp.employee_id ||' '|| v_emp.salary) ;  
                    END LOOP ;  
                    CLOSE v_emp_cursor ;  
                END;  
                /  
        # 取出工资超过 10000 的员工的 employee_id，FOR 方式  
            -   DECLARE  
                    CURSOR v_emp_cursor (a NUMBER) IS SELECT * FROM employees WHERE salary > a ;  
                BEGIN  
                    FOR i IN v_emp_cursor(10000) LOOP  
                    DBMS_OUTPUT.PUT_LINE(i.employee_id) ;  
                    END LOOP ;  
                END ;  
                /

知识点–概念：FOR 循环中使用显式游标和子查询

    1. 介绍
        # FOR 循环使用显式游标，会自动进行游标的 OPEN，FETCH，EXIT，CLOSE  
        # FOR 循环的计数变量 i 会自动声明类型  
        # FOR 循环如果使用子查询，自动以显式游标方式运行，不需要声明游标  
    2. 格式
        # 显示游标方式
            -   DECLARE  
                    CURSOR <cursor_name> IS <select_statement> ;  
                BEGIN  
                    FOR i IN <cursor_name> LOOP  
                        <statements> ;  
                    END LOOP ;  
        # 子查询方式
            -    BEGIN  
                FOR i IN (<sub_query>) LOOP  
                    <statements> ;  
                END LOOP ;

知识点–语法：FOR UPDATE，WHERE CURRENT OF，显式游标进行上锁

    1. 介绍  
        # 显式游标使用 FOR UPDATE 和 WHERE CURRENT OF，性能要比 UPDATE 使用关联子查询更好  
        # FOR UPDATE  
            - 在 OPEN 游标时，对基表的行加上独占锁  
            - 在 CLOSE 游标时，自动解锁  
        # WHERE CURRENT OF  
            - 使用游标更新基表，指定基表的行为当前游标的行  
    2.范例  
        # 使用游标更新 emp 表  
            -   DECLARE  
                    CURSOR c1 IS SELECT * FROM emp FOR UPDATE ;  
                    v_dept VARCHAR(20) ;  
                BEGIN  
                    FOR i IN c1 LOOP  
                        SELECT department_name INTO v_dept FROM departments WHERE department_id = i.department_id ;  
                        UPDATE emp SET dept_name = v_dept WHERE CURRENT OF c1 ;  
                    END LOOP ;  
                    COMMIT ;  
                END ;  
                /

21.4.4 动态游标

知识点–概念：动态游标，游标变量

    1. 静态游标  
        # 在 DECLARE 中就已经固定了所代表的 SELECT 语句  
        # 显式游标是静态游标  
    2. 动态游标  
        # 在 DECLARE 声明块中仅声明一个游标变量，不晓得 SELECT 语句是什么  
        # 在 BEGIN 执行块中使用 OPEN FOR 语句打开游标，顺便确定 SELECT 语句  
        # 在 BEGIN 执行块中，可以多次打开和关闭，并且每次的 SELECT 语句都不同  
    3. 三种动态游标  
        # SYS_REFCURSOR  
            - 弱类型，弱类型表示每次打开都可以打开为不同的数据类型  
            - 游标类型  
            - 可以作为函数或过程的 OUT 形参的数据类型  
        # REF CURSOR  
            - 弱类型，和 SYS_REFCURSOR 性质相同  
            - OPEN FOR 时可以使用动态打开方式，也可以使用静态打开方式  
        # REF CURSOR RETURN  
            - 强类型，强类型表示每次打开都只能打开固定的数据类型  
            - OPEN FOR 时只能用静态打开方式

知识点–语法：定义和声明动态游标

    1.REF CURSOR  
        # TYPE <type_name> IS REF CURSOR ;  
        # <cursor_name> <type_name> ;  
    2.SYS_REFCURSOR  
        # <cursor_name> SYS_REFCURSOR ;  
    3.REF CURSOR RETURN  
        # TYPE <type_name> IS REF CURSOR RETURN <data_type> ;  
        # <cursor_name> <type_name> ;

知识点–语法：OPEN FOR 打开动态游标

    1. 动态打开方式  
        # OPEN <cursor_name> FOR '<select_statement>' [USING <var_name>[, ...]] ;  
            - USING <var_name>[, ...]  
                在 <select_statement> 添加变量标记，在打开游标时指定变量值  
                变量标记的形式为 :<N>，在 USING <var_name>[, ...] 中按照位置进行变量值的替换  
                USING 中只能添加 <var_name>，不能添加常量  
    2. 静态打开方式  
        # OPEN <cursor_name> FOR <select_statement> ;  
    3. 范例  
        # 获取 salary 小于 avg(salary) 员工的 employee_id，使用 REF CURSOR，动态打开方式  
            - DECLARE  
                TYPE tc IS REF CURSOR ;  
                vc tc ;  
                TYPE tt IS TABLE OF employees%ROWTYPE INDEX BY PLS_INTEGER ;  
                vt tt ;  
                avg_sal NUMBER ;  
            BEGIN  
                SELECT AVG(salary) INTO avg_sal FROM employees ;  
                OPEN vc FOR 'SELECT * FROM employees WHERE salary < :1 ORDER BY salary' USING avg_sal ;  
                FETCH vc BULK COLLECT INTO vt ;  
                CLOSE vc ;  
                FOR i IN vt.FIRST()..vt.LAST() LOOP  
                    DBMS_OUTPUT.PUT_LINE(vt(i).employee_id);  
                END LOOP ;  
            END ;  
            /  
        # 获取 salary 小于 avg(salary) 员工的 employee_id，使用 SYS_REFCURSOR，动态打开方式  
            - DECLARE  
                vc SYS_REFCURSOR ;  
                TYPE tt IS TABLE OF employees%ROWTYPE INDEX BY PLS_INTEGER ;  
                vt tt ;  
                avg_sal NUMBER ;  
            BEGIN  
                SELECT AVG(salary) INTO avg_sal FROM employees ;  
                OPEN vc FOR 'SELECT * FROM employees WHERE salary < :1 ORDER BY salary' USING avg_sal ;  
                FETCH vc BULK COLLECT INTO vt ;  
                CLOSE vc ;  
                FOR i IN vt.FIRST()..vt.LAST() LOOP  
                    DBMS_OUTPUT.PUT_LINE(vt(i).employee_id);  
                END LOOP ;  
            END ;  
            /  
        # 获取 salary 小于 avg(salary) 员工的 employee_id，使用 REF CURSOR RETURN，静态打开方式  
            - DECLARE  
                TYPE tc IS REF CURSOR RETURN employees%ROWTYPE ;  
                vc tc ;  
                TYPE tt IS TABLE OF employees%ROWTYPE INDEX BY PLS_INTEGER ;  
                vt tt ;  
                avg_sal NUMBER ;  
            BEGIN  
                SELECT AVG(salary) INTO avg_sal FROM employees ;  
                OPEN vc FOR SELECT * FROM employees WHERE salary < avg_sal ORDER BY salary ;  
                FETCH vc BULK COLLECT INTO vt ;  
                CLOSE vc ;  
                FOR i IN vt.FIRST()..vt.LAST() LOOP  
                    DBMS_OUTPUT.PUT_LINE(vt(i).employee_id);  
                END LOOP ;  
            END ;  
            /

知识点–操作：动态游标不使用 BULK COLLECT

    1.将动态游标的数据保存到关联数组中  
        # 记录工资最低的 10 个人，LOOP 方式  
            - DECLARE  
                TYPE t_1 IS REF CURSOR ;  
                c_1 t_1 ;  
                TYPE t_2 IS TABLE OF employees%ROWTYPE INDEX BY PLS_INTEGER ;  
                v_1 t_2  ;   
                i NUMBER :=1 ;  
            BEGIN  
                OPEN c_1 FOR 'select * from employees FETCH FIRST 10 ROWS ONLY ORDER BY SALARY' ;  
                LOOP  
                    FETCH c_1 INTO v_1(i) ;  
                    EXIT WHEN C_1%NOTFOUND ;  
                    DBMS_OUTPUT.PUT_LINE( v_1(i).first_name||'  '||v_1(i).last_name || ' ' || v_1(i).salary ) ;  
                    i:= i+1 ;  
                END LOOP ;  
                CLOSE c_1 ;  
            END ;  
            /

21.5 异常处理

21.5.1 异常

知识点–概念：异常 exception

    1. 异常  
        # 程序运行过程中，发现错误 error，就会终止程序的运行，并抛出异常  
    2. 抛出异常 raise exception  
        # Implicitly Raised by Oracle 隐式抛出  
            - 即自动抛出  
            - 由 oracle 定义生么样的行为算是错误  
            - 由 Oracle 判断异常种类，异常编号，异常 message  
        # Explicitly Raised by Program 显式抛出  
            - 由用户定义什么样的行为算是错误，例如 UPDATE salary 规定涨幅不能超过 15%  
            - 由用户编写代码，确定抛出异常的位置，异常名称，异常编号，异常 message  
    3. 捕获异常 trapping exception  
        # 捕获异常的流程  
            - 添加和编写异常段  
            - 抛出异常后，如果异常的类型已经在异常段中被定义，则跳转到异常段中，执行异常段中的语句  
            - 结束异常段的执行后，结束程序 successfully  
        # 捕获异常的习惯  
            - 异常段中的代码，通常是打印和记录相应的错误信息  
            - 例如，将发生的异常记录到一个 error 表中，可以在程序调试阶段查看所有异常的出现频率  
        # 捕获异常和 COMMIT 和 ROLLBACK  
            - 如果 PLSQL 块中涉及 DML 时，最好在可能出现异常的 DML 语句后加 COMMIT，在 EXCEPTION 异常块的子句中加 ROLLBACK  
                DML 语句出现错误后，通过异常段自动回滚  
                DML 语句成功才能提交  
    4. 传播异常 propagating exception  
        # 抛出异常后，如果没有捕获到异常，则整个 PLSQL 程序块都会被终止  
        # 异常被传递给程序的调用环境  
        # 调用环境可以是 PLSQL 父块，可以是其他程序如 sqlplus  
        # 嵌套块环境对传播异常的处理  
            - 在子块中只编写此子块较常见的异常的处理  
            - 在父块中编写通用的异常处理  
    5. 捕获异常的优点  
        # 被捕获的异常不会影响到其他程序段的运行  
        # 可以在异常段的子句中添加 ROLLBACK，以防止异常影响数据的完整性  
        # 以下代码，虽然子块中发生了 too_many_rows 的异常，但子块中的异常被捕获，所以父块中的数据可以正常被打印  
            -   <<main_block>>  
                DECLARE  
                    v1 NUMBER ;  
                BEGIN  
                    SELECT salary INTO v1 FROM employees WHERE employee_id = 100 ;  
                    <<sub_block>>  
                    DECLARE  
                        v2 NUMBER ;  
                    BEGIN  
                        SELECT salary INTO v2 FROM employees WHERE salary > 10000 ;  
                    EXCEPTION  
                        WHEN too_many_rows THEN  
                            NULL ;  
                    END sub_block ;
                    DBMS_OUTPUT.PUT_LINE('v1 : '||v1) ;  
                END main_block ;
                /
    6. 异常的种类  
        # Predefined Oracle Server Error，PLSQL 预定义的异常类型，隐式抛出  
        # Non-Predefined Oracle Server Error，ORACLE 数据库 STANDARD ERROR，隐式抛出  
        # User-Defined Error，显式抛出  
    7. 常见的 Predefined Oracle Server Errors  
        # NO_DATE_FOUND  
        # TOO_MANY_ROWS  
        # INVALID_CURSOR  
        # ZERO_DIVIDE  
        # DUP_VAL_ON_INDEX

21.5.2 异常块

知识点–语法：EXCEPTION 异常块捕获异常

    1. 介绍
        # 对于被抛出的异常，编写代码确定如何进行处理  
    2. 异常块的习惯写法
        # 经常发生的异常应当放在前面，可以提高效率  
            - 因为捕获到异常后，PLSQL 只会执行一个异常处理子句，然后跳转到之后的代码继续运行  
    3. 语法  
        # EXCEPTION  
            WHEN <exception_name_1> [OR <exception_name_2> ...] THEN  
                <statements> ;  
            WHEN <exception_name_n> [OR <exception_name_m> ...] THEN  
                <statements> ;  
            ...  
            [WHEN OTHERS THEN <statements> ; ...]

知识点–函数：SQLCODE 返回错误代码，SQLERRM 返回错误信息

    1. 介绍
        # 通常用在异常块中的 WHEN OTHERS THEN 子句中，在调试代码阶段，查看错误代码和错误说明  
    2. 范例  
        # DECLARE  
            ... ;  
            v_error_code NUMBER ;  
            v_error_message VARCHAR(255) ;  
        BEGIN  
            ... ;  
        EXCEPTION  
            WHEN OTHERS THEN  
                v_error_code := SQLCODE ;  
                v_error_message :=SQLERRM ;  
                DBMS_OUTPUT.PUT_LINE( v_error_code) ;  
                DBMS_OUTPUT.PUT_LINE( v_error_message) ;  
        END ;  
        /

21.5.4 处理 User-Defined Error

知识点–语法：RAISE，显示抛出异常

    1.作用  
        # 用在 BEGIN 执行段的代码中，显式抛出一个已经声明过的异常，并跳转到异常段中  
        # 用在 EXCEPTION 异常段的子句中，则是强行将异常传播给调用环境  
    2.语法  
        # RAISE <exception_name> ;  
    3.范例  
        # 异常 no_salary_under_1000   
            - DECLARE  
                no_salary_under_1000 EXCEPTION ;  
            BEGIN  
                UPDATE employees SET salary = salary+1000 WHERE salary < 1000 ;  
                IF SQL%NOTFOUND THEN  
                    RAISE no_salary_under_1000 ;  
                END IF ;  
                COMMIT ;  
            EXCEPTION  
                WHEN no_salary_under_1000 THEN  
                    ROOLBACK ;  
                    DBMS_OUTPUT.PUT_LINE(' No one poor.') ;  
            END ;  
            /

知识点–过程：DBMS_STANDARD.RAISE_APPLICATION_ERROR，以 ORACLE 标准错误的形式显式抛出异常

    1.作用  
        # 显式抛出一个用户自定义的 ORA-<N> 的标准错误  
        # 用在 BEGIN 执行段的代码中，显式抛出一个异常  
        # 用在 EXCEPTION 异常段的子句中，则是将其他异常转换为 ORACLE 标准错误并传播给调用环境  
    2.格式  
        # RAISE_APPLICATION_ERROR ( NUM, MSG , KEEPERRORSTOCK)

            - MSG，显示的错误信息

    3.范例，  
        # 用在 BEGIN 执行段  
            - BEGIN  
                ... ;  
                UPDATE employees SET salary = salary+1000 WHERE salary < 1000 ;  
                IF SQL%NOTFOUND THEN  
                    RAISE_APPLICATION_ERROR( -20000, ' An error occured') ;  
                END IF ;  
            END ;  
            /  
        # 用在 EXCEPTION 异常段，将所有的异常都转换为 ORA-20001 错误  
            - BEGIN  
                ... ;  
            EXCEPTION  
                WHEN OTHERS THEN  
                    RAISE_APPLICATION_ERROR( -20001, 'An error occered') ;  
            END ;  
            /

21.6 动态 SQL

21.6.1 动态 SQL

知识点–概念：动态 SQL

    1.介绍  
        # 普通 SQL 是固定的  
        # 动态 SQL 是在编译时 SQL statement 不确定某些元素的值，在执行时才确定 SQL statement 的内容  
            - 如 WHERE condition 中比较条件的条件范围  
        # 动态 SQL 也可以运行动态 PLSQL 块  
            - 语句改为 'BEGIN <statements>; END ;' 的格式  
    2.动态 SQL 的优点  
        # 增加 PLSQL 程序的通用性和灵活性  
        # PLSQL 可以执行 DDL 操作

知识点–语法：EXECUTE IMMEDIATE，PLSQL 块中运行动态 SQL

    1.语法  
        # EXECUTE IMMEDIATE <string_var_name> [INTO {<defined_strings> [, ...] | RECORD}] [UNSING {[IN|OUT|IN OUT] <argument_name>}[, ...]] ;  
        # EXECUTE IMMEDIATE <dynamic_string> [INTO {<defined_strings> [, ...] | RECORD}] [UNSING {[IN|OUT|IN OUT] <argument_name>}[, ...]] ;  
    2. 语法说明
        # <string_var_name>  
            - 提前将需要执行的语句和动态的元素提前组合为长字符串，放在变量 <string_var_name> 中  
            - 通过 <string_var_name> 执行语句  
        # <dynamic_string>  
            - 直接在 EXECUTE IMMEDIATE 中组合需要执行的语句  
        # [INTO {<defined_strings> [, ...] | RECORD}]  
            - 将查询结果保存到某处  
            - 只能用在返回一行的查询中  
        # [USING {[IN|OUT|IN OUT] <argument_name>}[, ...]]  
            - 进行变量值的替换  
            - 在 <string_var_name> 或 <dynamic_string> 中，使用 :<p_name> 的方式标记替换位置  
            - 在 UNSING 中指定替换相应的 <p_name> 使用的 <var_name> 或 <value>  
            - 替换只能按照位置进行  
    3.动态 SQL 执行 DDL  
        # 不能使用 USING 进行替换  
    4.范例  
        # 动态 PLSQL  
            -   BEGIN  
                    EXECUTE IMMEDIATE 'BEGIN DBMS_OUTPUT.PUT_LINE(321) ;END ;' ;  
                END ;  
                /  
        # 创建过程 p_create_t，可以进行建表  
            -   CREATE OR REPLACE PROCEDURE p_create_t ( p_table_name VARCHAR2, p_col_spec VARCHAR2)  
                    IS  
                        v_xxxx VARCHAR2(1024) ;  
                    BEGIN  
                        -- v_xxxx := 'CREATE TABLE ' || p_table_name  || ' ( ' || p_col_spec || ' ) ' ;  
                        EXECUTE IMMEDIATE 'CREATE TABLE ' || p_table_name  || ' ( ' || p_col_spec || ' ) ' ;  
                    END p_create_t;  
                    /  
            -   EXEC p_create_t( 't003' , ' employee_id number PRIMARY KEY, employee_name VARCHAR(30) NOT NULL, salary NUMBER NOT NULL') ;

21.7 批量绑定

21.7.1 批量绑定 Bulk Binds

知识点–概念：PLSQL 和 sql 之间的切换

    1. ORACLE 处理 PLSQL 程序块需要两个引擎  
        # PLSQL 引擎，运行 PLSQL 语句，将 sql 语句传递给 sql 引擎  
        # sql 引擎，运行 sql 语句，将结果返回给 PLSQL 引擎  
    2. 程序块中的每一个 sql 语句，都会引起两个引擎的环境切换，可能影响性能  
        # 例如 FOR 循环中使用 SELECT INTO 语句进行对 collection 类型变量进行赋值，每一轮循环都要切换一次环境  
    3. 通过使用批量绑定，可以减少引擎切换的次数

知识点–概念：批量绑定

    1. 批量绑定的效果，将多次切换才能进行的数据传递只用一次切换来完成  
    2. 批量绑定的两种方式及其应用场景  
        # FORALL 语句
            - 增强 PLSQL 引擎到 sql 引擎
            - 引用 collection 变量的内容来 INSERT or UPDATE or delete 表  
        # BULK COLLECTION INTO 语句
            - 增强 sql 引擎到 PLSQL 引擎
            - 引用 table 的内容对 collection 变量进行大批量的赋值

21.7.4 FORALL

知识点–概念：FORALL 批量绑定 DML

    1.对一行 DML 语句，采用批量绑定的方式进行数据的修改

知识点–语法：FORALL

    1. 语法  
        # FORALL i IN <lower>..<upper> [SAVE EXCEPTION] <dml_statement> ;  
            - SAVE EXCEPTION，将 FORALL 中的异常保存到 SQL%BULK_EXCEPTIONS 游标中  
    2. FORALL INSERT  
        # 可以使用数组项作为 INSERT 语句的 value 值  
            - 例如，FORALL i IN <lower>..<upper> SAVE EXCEPTION INSERT INTO <table_name> VALUES V01(i) ;  
        # 可以使用数组项中的列作为 INSERT 语句的 value 值  
            - 例如，FORALL i IN <lower>..<upper> SAVE EXCEPTION INSERT INTO <table_name> VALUES (v01(i).employee_id + v01(i).department_id ,v01(i).salary) ;  
    3.FORALL UPDATE  
        # FORALL 的索引可以出现在 WHERE 子句中，但不能出现在 SET 子句中  
            - 例如，FORALL i IN 1..v01.last SAVE EXCEPTIONS UPDATE testemp SET salary = salary +10000 WHERE employee_id = v01(i).employee_id ;  
            - 语句失败，FORALL i IN 1..v01.last SAVE EXCEPTIONS UPDATE testemp SET salary = v01(i).salaey WHERE employee_id = v01(i).employee_id ;  
    4.FORALL DELETE  
        # FORALL 的索引用在 WHERE 子句中

知识点–概念：SQL%BULK_ROWCOUNT 隐式游标

    1.储存 FORALL 语句中的每一次操作所影响的行数  
        # SQL%BULK_ROWCOUNT(i) 就等于第 i 次操作所修改的行的数量  
    2.范例  
        # 将所有 department_id 小于 49 的部门的员工，工资上调 10000 元，然后查看 FORALL 每次修改的行数  
            -   DECLARE  
                    CURSOR c01 (n NUMBER) IS (SELECT DISTINCT department_id FROM employees WHERE department_id < n) ;  
                    TYPE t01 IS TABLE OF c01%ROWTYPE INDEX BY PLS_INTEGER ;  
                    v01 t01 ;  
                    zv01 NUMBER := 49 ;  
                BEGIN  
                    OPEN c01(zv01) ;  
                    FETCH c01 BULK COLLECT INTO v01 ;  
                    CLOSE c01 ;  
                    -- FORALL 修改  
                    FORALL i in 1..v01.LAST SAVE EXCEPTIONS  
                        UPDATE testemp SET salary = salary + 10000 WHERE department_id = v01(i).department_id ;  
                    -- 查看每次修改的行数  
                    FOR i IN 1..v01.LAST LOOP  
                        DBMS_OUTPUT.PUT_LINE(SQL%BULK_ROWCOUNT(i)) ;  
                    END LOOP ;  
                END ;  
                /

知识点–概念：SQL%BULK_EXCEPTIONS 隐式游标

    1.默认情况下，当 FORALL 语句出错，ORACLE 会中断语句的执行并抛出异常  
        # 已执行的操作不会回退  
        # 未执行的操作不再执行  
    2.当指定了 SAVE EXCEPTIONS 后，FORALL 语句中出现的异常会放到 SQL%BULK_EXCEPTIONS 隐式游标中  
        # 所有的操作全部执行  
        # 成功的操作保存  
        # 失败的操作，将错误信息保存到 SQL%BULK_EXCEPTIONS 中  
    3.FORALL 语句抛出的异常为 ORA-24381  
        # SAVE EXCEPTIONS 但不捕获异常，则 ORACLE 在 FORALL 语句执行完成后再抛出异常  
        # SAVE EXCEPTIONS 并捕获异常，则 ORACLE 在 FORALL 语句执行完成后转到异常段执行  
    4.SQL%BULK_EXCEPTIONS 的属性  
        # SQL%BULK_EXCEPTIONS.COUNT                 <-- 隐式游标 SQL%BULK_EXCEPTIONS 中记录的信息的个数  
        # SQL%BULK_EXCEPTIONS(i).ERROR_INDEX        <-- 第 i 个错误记录的是 FORALL 中第几次操作
        # SQL%BULK_EXCEPTIONS(i).ERROR_CODE         <-- 第 i 个错误记录的错误代码  
    5.范例  
        # 使用 BULK COLLECT INTO 和 FORALL，将 salary 小于平均值的员工的 employee_id 记录下来  
            -   CREATE TABLE test (id NUMBER) ;     <-- 创建结果表  
            -   DECLARE  
                    CURSOR c01 (avg_sal NUMBER) IS (SELECT * FROM employees WHERE salary < avg_sal) ;  
                    TYPE t01 IS TABLE OF c01%ROWTYPE INDEX BY PLS_INTEGER ;  
                    v01 t01 ;  
                    zv01 NUMBER ;  
                BEGIN  
                    SELECT avg(salary) INTO zv01 FROM employees ;  
                    -- DBMS_OUTPUT.PUT_LINE(zv01) ;  
                    -- FETCH <data> FROM <cursor>  
                    OPEN c01( zv01) ;  
                    FETCH c01 BULK COLLECT INTO v01 ;  
                    CLOSE c01 ;  
                    -- FORALL INSERT  
                    FORALL i IN 1..v01.LAST SAVE EXCEPTIONS  
                        INSERT INTO test VALUES (v01(i).employee_id ) ;  
                EXCEPTION  
                    WHEN OTHERS THEN  
                        FOR i IN 1..SQL%BULK_EXCEPTIONS.COUNT LOOP  
                            DBMS_OUTPUT.PUT_LINE( 'ERRORS==>>'||SQL%BULK_EXCEPTIONS(i).ERROR_INDEX||' '|| SQL%BULK_EXCEPTIONS(i).ERROR_CODE) ;  
                        END LOOP ;  
                END ;  
                /

21.7.4 RETURNING BULK COLLECT INTO

知识点–语法：RETURNING BULK COLLECT INTO

    1.将修改后的行的每一行的一部分内容记录下来  
        # INSERT 和 UPDATE，记录修改后的内容  
        # DELETE，记录修改前的内容  
    2.范例  
        # 使用 FETCH BULK COLLECT INTO，将 salary 小于平均值的员工的 employee_id 记录到 v01 关联数组中  
        # 使用 FORALL，将这些员工的 salary 加 10000  
        # 使用 RETURNING BULK COLLECT INTO，将上一步的修改的员工的 employee_id 和修改后的 salary 分别记录到 v02 和 v03 关联数组中  
            -   CREATE TABLE testemp AS SELECT * FROM employees ;           -- 创建测试表  
            -   DECLARE  
                    CURSOR c01 (avg_sal NUMBER) IS (SELECT * FROM testemp WHERE salary < avg_sal) ;  
                    TYPE t01 IS TABLE OF c01%ROWTYPE INDEX BY PLS_INTEGER ;  
                    v01 t01 ;  
                    TYPE t02 IS TABLE OF NUMBER INDEX BY PLS_INTEGER ;  
                    v02 t02 ;  
                    v03 t02 ;  
                    zv01 NUMBER ;  
                BEGIN  
                    SELECT AVG(salary) INTO zv01 FROM employees ;  
                    DBMS_OUTPUT.PUT_LINE(zv01) ;  
                    -- FETCH <data> FROM <cursor>  
                    OPEN c01( zv01) ;  
                    FETCH c01 BULK COLLECT INTO v01 ;  
                    CLOSE c01 ;  
                    -- FORALL UPDATE 和 RETURNING  
                    FORALL i IN 1..v01.last SAVE EXCEPTIONS  
                        UPDATE testemp SET salary = salary + 10000  
                            -- delete from testemp  
                            WHERE employee_id = v01(i).employee_id  
                            RETURNING employee_id, salary BULK COLLECT INTO v02,v03 ;  
                    FOR i IN 1..v02.last LOOP  
                        DBMS_OUTPUT.PUT_LINE(v02(i) ||' '|| v03(i)) ;  
                    END LOOP ;  
                EXCEPTION  
                    WHEN OTHERS THEN  
                        FOR i IN 1..SQL%BULK_EXCEPTIONS.COUNT LOOP  
                            DBMS_OUTPUT.PUT_LINE( 'ERRORS==>>'||SQL%BULK_EXCEPTIONS(i).ERROR_INDEX||' '|| SQL%BULK_EXCEPTIONS(i).ERROR_CODE) ;  
                        END LOOP ;  
                END ;  
                /

第22章 PLSQL 程序

22.1 参数

22.1.1 参数介绍

知识点–概念：PLSQL 程序参数

    1.介绍  
        # 用来在程序和调用环境之间传递数据  
        # 字符型函数不能使用 VARCHAR2(<n>)，只能使用 VARCHAR2  
    2.形参 formal parameter  
    3.实参 actual parameter

知识点–概念：参数的三种模式，IN，OUT，IN OUT

    1.IN 参数  
        # 从调用环境传递一个数值给程序  
        # 在执行中不能进行赋值  
        # 在执行段中，IN 参数作为一个只读的常数而存在  
    2.OUT 参数  
        # 从程序传递一个数值给调用环境  
        # 必须传递实参  
        # 实参必须是一个变量  
        # 不能设置默认值  
        # sqlplus 环境通过绑定变量获得参数值  
        # PLSQL 程序块中将父块中的变量作为实参，继而获得参数值  
    3.IN OUT 参数  
        # 程序执行前从调用环境传递一个数值给程序，程序执行结束后又从程序传递一个数值给调用环境  
        # 必须传递实参  
        # 实参必须是一个变量  
        # 不能设置默认值

22.1.2 传递参数值

知识点–概念：传递参数的方法

    1.按位置传递 positional  
        # 按照过程的定义中的形参顺序填写实参  
        # 要求实参所对应的要连续  
        # 如果实参数量少于形参数量，则实参按顺序赋予前半部分的形参，后半部分的形参使用默认值  
        # EXEC <procedure_name>( <value_1>, <value_2>, ...) ;  
    2.按名称传递 named  
        # 按照 <parameter_name> => <value> 的形式传递实参，不需要注意顺序  
        # 未指定实参的形参使用默认值  
        # EXEC <procedure_name>( <parameter_name_x> => <value_x>, <parameter_name_y> => <value_y>, ...) ;  
    3.组合方式 combination  
        # 某些参数按位置传递，某些参数按名称传递  
        # 按位置传递要放在前，按名称传递放在后并且最好有默认值  
        # EXEC <procedure_name>( <value_1>, <value2>, <parameter_name_x> => <value_x>, <parameter_name_y> => <value_y>, ...)

知识点–概念：定义过程时参数的推荐先后顺序

    1.OUT 和 IN OUT 参数放在前，较常修改的 IN 参数放在前  
        # OUT 和 IN OUT 参数必须指定实参，故放在前，可以使用按位置传递  
        # 较常修改的 IN 参数放在前，可以使用按位置传递  
    2.不常用的 IN 参数放在后，并指定默认值  
        # 当没有指定时，使用默认值  
        # 当需要指定时，写在按位置实参之后，使用按名称方式传递

22.2 过程

22.2.1 PLSQL 子程序

知识点–概念：子程序 subprogram

    1.一个命名的 PLSQL 程序块，可以是过程，也可以是函数  
    2.子程序包含，一个声明段，一个执行段，一个异常段  
    3.子程序创建后，作为对象保存在数据库中，然后自动进行编译，编译后，保存在数据库中  
        # 未编译或编译失败的子程序，在 DBA_OBJECTS 中，STATUS 列的状态为 INVALID，在 DBA_ERRORS 中记录了相关的编译错误  
    4.字典  
        # DBA_PROCEDURES            <-- 记录所有的过程  
        # DBA_TRIGGERS              <-- 记录所有的触发器  
        # DBA_SOURCE                <-- 记录所有的存储的对象的源码  
        # DBA_ERRORS                <-- 记录当前数据库中储存过程的编译错误

知识点–概念：模块化编程

    1.将大的程序代码块转换成小的称之为模块的程序块  
    2.分层的程序设计  
        # 数据访问层 data access layer，使用 SQL 语句访问数据的子程序  
        # 业务逻辑层 business logic layer

22.2.2 过程 Procedure

知识点–概念：过程 Procedure

    1.过程通常只进行操作  
    2.调用过程的方式  
        # sqlplus 环境中，可以使用 EXEC 语句调用过程  
        # sqlplus 环境中，可以使用匿名块调用过程  
        # 其他 PLSQL 子程序中，可以直接使用 <procedure_name> 调用过程  
            - 在过程的声明段中，可以定义过程  
            - 在过程的执行段中，调用其他过程，需要保证其他过程已经被编译并保存

知识点–语法：CREATE PROCEDURE 创建过程

    1.语法  
        # CREATE [OR REPLACE] PROCEDURE <procedure_name>  
            [({<parameter_name> [IN|OUT|IN OUT] <data_type> [DEFAULT <default_value>] [:= <value>]}[, ...] )]  
            IS|AS  
                [<locat_variable_declearations> ;]  
            BEGIN  
                <statements> ;  
            END [<procedure_name>] ;  
            /

知识点–语法：ALTER PROCEDURE COMPILE 重新编译过程

    1.语法  
        # ALTER PROCEDURE <procedure_name> COMPILE ;

知识点–语法：DROP PROCEDURE 删除过程

    1.语法  
        # DROP PROCEDURE <procedure_name> ;

22.2.3 过程和异常

知识点–概念：过程中的异常处理

    1.过程中抛出异常  
        # 未提交的事务被回退  
        # 对绑定变量的修改被回退  
    2.过程中抛出异常并被捕获  
        # 未提交的事务保持不变  
        # 对绑定变量的修改保存

知识点–概念：父过程调用子过程时的异常处理

    1.父过程调用子过程，子过程抛出异常  
        # 父过程和子过程中所有未提交的事务都回退  
        # 父过程和子过程中所有对绑定变量的修改全部回退  
    2.父过程调用子过程，子过程捕获异常  
        # 子过程中异常发生后的操作不再进行  
        # 父过程中异常发生后的操作正常进行  
        # 父过程和子过程中所有未提交的事务都以未提交状态保持不变  
        # 父过程和子过程中所有对绑定变量的修改都保存  
    3.父过程调用子过程，子过程抛出异常，父过程捕获异常  
        # 父过程中在子过程之后的操作不再进行  
        # 子过程中未提交的事务全部回退  
        # 父过程中未提交的事务保持不变  
        # 子过程中涉及到的绑定变量被清空  
        # 父过程中对绑定变量的修改保存

22.3 函数

22.3.1 函数

知识点–概念：函数 Function

    1.函数主要用来进行计算，返回一个值  
        # 很少用函数进行操作  
    2.函数只能有一个返回值  
        # 在执行块中，可以有多个 RETURN，但有且只能有一个 RETURN 子句被执行  
        # 函数最好不要有 OUT 和 IN OUT 的参数，因为这样的函数有多个出口，不利于维护  
        # 如果必须获得多个出口，可以使用 PLSQL 的组合数据类型  
    3.函数接收参数，可以是按位置，也可以是按名称  
        # 在 11g 之前，不接受按名称  
    4.函数中最好添加异常块，防止因为没有返回值而抛出异常

知识点–语法：CREATE FUNCTION 创建函数

    1.语法  
        # CREATE [OR REPLACE] FUNCTION <function_name>  
            [({<parameter> [IN|OUT|IN OUT] <datatype> [DEFAULT <default_value>] [:= <value>]}[, ...] )]  
            RETURN <datatype>  
            IS|AS  
                [<local_variable_declearations> ;]  
            BEGIN  
                <statements> ;  
                RETURN <expression> ;  
            END [<function_name>] ;  
            /  
    2.范例  
        # 输入两个数，输出较大的那个数  
            -   CREATE OR REPLACE FUNCTION f1 ( p1 NUMBER, p2 NUMBER)  
                    RETURN NUMBER  
                    IS  
                        p3 NUMBER ;  
                    BEGIN  
                        IF p1 >= p2 THEN p3 := p1 ;  
                            ELSE p3 := p2 ;  
                        END if ;  
                        RETURN p3 ;  
                    END f1 ;

知识点–语法：DROP FUNCTION 删除函数

    1.语法  
        # DROP FUNCTION <function_name> ;

22.3.2 函数的使用

知识点–概念：调用函数所需要的权限

    1.OWNER of the function  
    2.EXECUTE on the function

知识点–概念：调用函数的方法

    1.放在 SELECT 等 SQL 语句中  
        # 例子  
            - SELECT function_1(n) FROM ... ;  
    2.在 sqlplus 环境使用 EXECUTE 调用，使用绑定变量获得函数的结果  
        # 例子  
            - VARIABLE var_1 NUMBER ;  
            - EXECUTE :var_1= funcation_1( n) ;  
    3.在 sqlplus 环境使用匿名块调用，使用绑定变量获得函数的结果  
    4.由 PLSQL 父环境调用，使用父环境 PLSQL 变量获得函数的结果  
        # 例子  
            - DECLARE  
                v_1 NUMBER ;  
            BEGIN  
                v_1 := function_1( n) ;  
                DBMS_OUTPUT.PUT_LINE( v_1) ;  
            END ;  
            /  
    5.放在其他子程序中调用，作为 IN 实参  
        # 例子  
            - DECLARE  
                v_1 NUMBER ;  
            BEGIN  
                v_1 := <sub_program_x>(<function_1>(<n>)) ;  
                DBMS_OUTPUT.PUT_LINE( v_1) ;  
            END ;  
            /

知识点–概念：函数在 SQL 语句中可以出现的位置

    1.select_list  
    2.where_condition，having_condition  
    3.GROUP BY 和 ORDER BY 和 CONNECT BY 和 START WITH  
    4.INSERT VALUES  
    5.UPDATE SET

知识点–概念：SQL 调用函数的限制

    1.函数的参数只能有 IN 参数，参数的数据类型必须是 SQL 支持的数据类型，返回值也必须是 SQL 支持的数据类型  
    2.函数中不能包含 COMMIT 和 ROLLBACK  
    3.函数不可以用在 CREATE TABLE 和 ALTER TABLE 的 CHECK 约束上  
    4.函数不可以用在 CREATE TABLE 和 ALTER TABLE 的 DEFAULT 上  
    5.函数不能包含 DML  
    6.在 UPDATE 和 DELETE 中，函数不能对基表进行查询或修改

22.3.3 RETURN 语句

知识点–语法：RETURN 语句

    1.用来退出程序块的执行并将控制权返回给调用环境  
        # 函数中，RETURN 同时将计算结果返回给调用环境  
        # 匿名块和过程中，RETURN 只是将控制权返回给调用环境  
        # 循环中，RETURN 退出循环并且退出程序块，因而会跳过循环块之后的所有语句  
    2.一个程序块中可以包含多个 RETURN 语句  
    3.语法  
        # RETURN ;  
        # RETURN <expression> ;

22.4 包

22.4.1 包

知识点–概念：包 Package

    1.包中包含 PLSQL 类型，变量，异常，子程序  
    2.包包括两个部分  
        # 包头 specification  
        # 包体 body  
        # 包头和包体必须名字相同  
    3.空间换时间  
        # 一个包中的一个内容被引用时，整个包都会加载到内存中  
        # 内存中的软件包被所有用户所共享  
        # 无体包  
            - 即没有包体的包  
            - 通常是用来声明常用的数据类型或常量或变量或异常，使这些内容常驻内存  
    4.子程序重载 overload  
        # 包中的子程序可以同名  
    5.包的公共组件和私有组件  
        # 公共组件可以调用公共组件和私有组件  
        # 私有组件可以调用公共组件和私有组件  
        # 外部程序可以调用公共组件，无法调用私有组件  
    6.包头 package specification  
        # 包头中的内容都是公共组件，作为应用程序的接口  
        # 可包含内容  
            - 数据类型，变量，常量，异常，游标  
            - 公共子程序的说明 sub_programs specifications  
            - 编译器指令 PRAGMAs  
    7.包体 package body  
        # 包体中的内容都是私有组件  
        # 可包含内容  
            - 数据类型，变量，常量，异常，游标  
            - 公共子程序的代码  
            - 私有子程序  
        # 必须先创建包头，再创建包体  
        # 改变包体，不需要重新编译包头  
        # 包体中，一个组件应该先定义，再被调用  
            - 通常，首先定义私有变量和私有子程序，最后定义公共子程序

22.4.2 创建包

知识点–语法：CREATE PACKAGE 创建包头

    1.语法  
        # CREATE [OR REPLACE] PACKAGE <package_name>  
            IS|AS  
                <public_type_and_variable_declarations> ;  
                <sub_programs_specification>[ ... ]
            END [<package_name>] ;  
            /  
        # <sub_programs_specification>  
            - 过程，PROCEDURE <procedure_name>(<parameters>) ;  
            - 函数，FUNCTION <function_name>(<parameters>) RETURN <data_type> ;

知识点–语法：CREATE PACKAGE BODY 创建包体

    1. 语法  
        # CREATE [OR REPLACE] PACKAGE BODY <package_name>  
            IS|AS  
                <privite_type_and_variable_declarations> ;  
                <sub_program body>[ ... ] 
                [BEGIN <initialization_statements> END ;]  
            END [package_name] ;  
            /  
        # <sub_program bodys>  
            - 过程  
                PROCEDURE <procedure_name>(<parameters>)   
                    IS   
                        <local_var_declarations> ;  
                    BEGIN  
                        <STATEMENTS> ;  
                    END <procedure_name> ;  
            - 函数  
                FUNCTION <function_name>(<parameters>) RETURN <data_type>   
                    IS  
                        <local_var_declarations> ;  
                    BEGIN  
                        <STATEMENTS> ;  
                    END <function_name> ;  
    2. 范例
        # 创建一个 salary_pkg 包  
            - 一个公共过程 reset_salary，输入员工的工资和职位，查看工资是否合规  
            - 一个私有函数 validate_salary，用以检查工资是否合规  
            - 公共过程 reset_salary 中需要调用私有函数 validate_salary  
        # 代码
            - CREATE OR REPLACE PACKAGE salary_pkg  
                IS  
                    v_std_salary NUMBER := 1000 ;                                                       <-- 初始化工资为 1000  
                    PROCEDURE reset_salary( emp_salary NUMBER, emp_job_id VARCHAR2 ) ;                  <-- 公共过程 reset_salary，检查工资是否处于工资范围  
                END salary_pkg ;                                                                        <-- 包头中不包含私有组件 validate_salary
                /  
            - CREATE OR REPLACE PACLAGE BODY salary_pkg  
                IS  
                    FUNCTION validate_salary( p_sal NUMBER, p_job_id VARCHAR2(20)) RETURN BOOLEAN       <-- 私有子程序 validate_salary，进行工资合规的判断  
                        IS  
                            v_min_sal jobs.min_salary%type ;  
                            v_max_sal jobs.max_salary%type ;  
                        BEGIN  
                            SELECT min_salary, max_salary INTO v_min_sal, v_max_sal FROM jobs WHERE job_id = p_job_id ;  
                            RETURN( p_sal BETWEEN v_min_sal AND v_max_sal ) ;  
                        END validate_salary ;  
                    PROCEDURE reset_salary( emp_salary NUMBER, emp_job_id VARCHAR2(20))  
                        IS  
                        BEGIN  
                            IF validate_salary( emp_salary, emp_job_id) THEN  
                                DMBS_OUTPUT.PUT_LINE(' Salary Allowed.') ;  
                            ELSE  
                                DMBS_OUTPUT.PUT_LINE(' Salary Not Allowed.') ;  
                            END IF ;  
                        END reset_salary ;  
                END salary_pkg ;  
                /

22.4.3 包的其他特性

知识点–概念：子程序重载 overload

    1.一个包下的子程序可以重名，叫做子程序的重载  
        # 没有放在包含在包中的独立的过程和函数还是不能重名  
    2.子程序的重载优点  
        # 子程序可以接收不同数据类型和数量的形参  
        # 在不改变源代码的情况下扩展子程序的功能  
    3.不可重载的情况  
        # 两个子程序的参数，只有数据类型不同，并且数据类型属于同一大类如都是数值型，或属于同一子类型例如 VARCHAR 和 STRING  
        # 两个子程序的参数相同，只有返回值的类型不同  
    4.字典
        # DBA_PROCEDURES.overload                   <-- 记录了重载的子程序在包中的重载的编号

知识点–概念：向前声明 Forward Declarations

    1.向前声明为了解决私有子程序的先声明后引用的问题  
        # 一般子程序的声明顺序是按照字母排列  
        # 如果前面的子程序用到了之后声明的子程序，就会报错  
    2.常用到向前声明的情况  
        # 按照逻辑或字母顺序定义子程序  
        # 两个子程序相互递归调用  
    3.向前声明的方式
        # 在包体的最前方添加子程序的说明 specifications  
        # 范例
            - procedure_2 调用到了 procedure_1，但是 procedure_1 的定义在 procedure_2 之后  
            - CREATE OR REPLACE PACKAGE BODY package_name  
                IS  
                    PROCEDURE procedure_1 ( parameters ) ;  
                    ... ;  
                    PRECEDURE procedure_2 ( parameters ) IS  
                        BEGIN  
                            statements ;  
                            exec_procedure_1_statements ;  
                            statements ;  
                        END procedure_2 ;  
                    ... ;  
                    PROCEDURE procedure_1 ( parameters ) IS  
                        BEGIN  
                            statements ;  
                        END [procedure_1] ;  
                    ... ;  
                END package_name ;  
                /

知识点–概念：包的初始化变量

    1.加载包时，未初始化的变量自动为 null  
    2.对于简单的情况，可以变量的声明中使用 DEFAULT 或 := 进行赋值  
    3.对于复杂的情况，可以在 PACKAGE BODY 的末尾追加初始化代码块 <initialization_statements>，称之为一次性过程  
        # 如果对一个变量多次初始化，则以最后一次初始化为准  
        # 范例  
            - CREATE OR REPLACE PACKAGE BODY <package_name>  
                IS  
                    ... ;  
                BEGIN  
                    var_1 := value_1 ;  
                    SELECT value_2 INTO var_2 FROM table ;  
                    ... ;  
                END <package_name> ;  
                /

知识点–概念：在 SQL 中使用包中的函数的限制

    1.在 SQL 中使用 package.function，需要评价函数的纯净度 purity level，即考察函数的副作用 side effect  
        # 所谓副作用，就是对数据库中数据的修改，对公共变量的修改  
        # 副作用越少，则 SQL 越容易优化，特别是指定的 PARALLEL_ENABLE 和 DETERMINISTIC 时  
    2.纯净度的要求  
        # 在一个 SELECT 或并行 DML 中，函数不能修改数据库中任何表的数据  
        # 在一个 DML 中，函数不能查询也不能修改当前基表  
        # 在一个 SELECT 或 DML 中，函数不能执行 transaction control 或 session control 或 system control 语句  
    3.任何违反纯净度的 SQL 语句，都会报错

22.4.4 包的持续状态

知识点–概念：包的持续状态 persistent state

    1.包中所有公共变量和私有变量的值，代表包的状态  
    2.默认情况下，包的状态与用户会话期相关联，包的状态是连续的  
        # 包的公共变量和私有变量的值保存在 UGA 中  
        # 每一个会话的包的状态都由自己所私享，不同会话之间互不干扰  
        # 每一个会话的变量，保存的是上一次调用的结果  
        # 在整个会话期，包的状态是连续的  
        # 对于游标，整个会话期若没有显式关闭游标，则游标一直处于打开状态并记录上次调用结束的位置  
    3.串行可重用包  
        # 对于串行可重用包，包的状态是不连续的  
        # 包的状态保存在 SGA 中  
        # 每次调用包，变量便被初始化  
        # 调用期间，包的变量仅保存此次调用的上一个执行周期的结果  
        # 调用结束，则包的变量值清空  
        # 对于游标，一次调用结束就会自动关闭游标  
    4.串行可重用的优点  
        # 对于包的运行需要大量临时工作区和临时变量，但都是只用一次，指定串行可重用可以节省内存空间

知识点–语法：PRAGMA SERIALLY_REUSABLE，创建串行可重用包

    1.在包的代码中，指定 PRAGMA SERIALLY_REUSABLE，表示包为串行可重用包  
        # 无体包，只需在包头中指定  
        # 有体包，要同时在包头和包体中指定  
    2.范例  
        # 有体包  
            - CREATE OR REPLACE PACKAGE funtest  
                IS  
                    PRAGMA SERIALLY_REUSABLE ;  
                    var_test NUMBER DEFAULT 1 ;  
                    PROCEDURE p_test ;  
                END funtest ;  
                /  
            - CREATE OR REPLACE PACKAGE BODY funtest  
                IS  
                    PRAGMA SERIALLY_REUSABLE ;  
                    PROCEDURE p_test IS  
                        BEGIN  
                            v_test := v_test +1 ;  
                            DBMS_OUTPUT.PUT_LINE( v_test) ;  
                        END p_test ;  
                END funtest ;  
                /

22.6.1 触发器 Trigger

知识点–概念：触发器 trigger

    1.触发一个触发器 fire a trigger  
    2.触发器的级别  
        # ON <table_name>  
        # ON <view_name>  
        # ON <user>.<schema>  
        # ON DATABASE  
    3.触发事件  
        # DML 事件  
            - INSERT  
            - UPDATE  
            - UPDATE OF <column_name>  
            - DELETE  
        # DDL 事件，CREATE，ALTER，DROP  
        # 数据库事件，SERVERERROR，LOGON，LOGOFF，STARTUP，SHUTDOWN  
    4.触发器的类型  
        # DML 触发器  
            - BEFORE  
            - AFTER，必须要在 COMMIT 之后，才算作触发  
            - INSTEAD OF，通常用于对视图的修改  
        # 复合触发器 Compound Trigger  
        # DDL 触发器  
        # Database event 触发器  
    5.触发器的级别  
        # 行级触发器 row-level trigger  
            - 使用 FOR EACH ROW 表示行级触发器

        # 语句级触发器 statement-level trigger  
            - 只针对语句触发一次，即使无返回行也会触发  
        # 事件级触发器 event-level trigger  
            - 针对数据库触发一次  
    6.数据库触发器和应用触发器  
        # database trigger，数据库范围内只要发生相关事件，就触发  
        # application trigger，只在应用范围内进行触发  
    7.触发器常用的作用  
        # 安全 Security  
        # 审计 Auditing  
        # 数据完整性 Data integrity  
        # 引用完整性 Referential integrity  
        # 表复制 replication  
        # 导出数据 Computing derived data automatically  
        # 事件记录 Event logging  
    8.触发器的限制  
        # 触发器中不可以进行 COMMIT 和 ROLLBACK  
    9.字典  
        # DBA_TRIGGERS              <-- 记录所有的触发器

知识点–概念：创建触发器所需要的权限

    1.CREATE TRIGGER 系统权限  
    2.ALTER TABLE 对象权限，或者 ALTER ANY TABLE 系统权限  
    3.对于数据库级别的触发器，需要 ADMINISTER DATABASE TRIGGER 系统权限

知识点–操作：创建 pdb 自动启动触发器

    1. 代码
        #   CREATE OR REPLACE TRIGGER sys.<trigger_name> AFTER STARTUP ON DATABASE

22.6.2 DML 触发器

知识点–概念：各种 DML 触发器的触发时间点

    1.触发时间点  
        # DML 语句级触发器 BEFORE 型  
        # DML 行级触发器 BEFORE 型  
        # 执行 DML 操作  
        # DML 行级触发器 AFTER 型  
        # DML 语句级触发器 AFTER 型  
        # COMMIT | ROLLBACK  
    2.如果 DML 语句执行失败，或触发器抛出异常并没有被捕获，则回退此条语句，之前的未提交的修改保持不变

知识点–语法：CREATE TRIGGER 创建 DML 行级触发器和语句级触发器

    1.语法  
        # CREATE [OR REPLACE] TRIGGER <trigger_name>  
            <timing> <event> [ OR <event> OR <event> ...]  
            ON <object_name>  
            [REFERENCING [OLD AS <old_val_name>] [NEW AS <new_val_name>]]  
            [FOR EACH ROW [WHEN (<condition>)]]  
            [DISABLE]  
            [DECLARE]  
            BEGIN  
                <trigger_body> ;  
            [EXCEPTION WHEN...THEN WHEN...THEN]  
            END [<trigger_name>] ;  
            /  
        # FOR EACH ROW [WHEN (<condition>)]   
            - 指定则为行级触发器  
            - 省略则为语句级触发器  
            - WHEN (<condition>)，对行级触发器，检查每一行是否执行 trigger body，括号是必须的  
        # <timing>  
            - 触发器的触发时机  
            - BEFORE  
            - AFTER  
            - INSTEAD OF  
        # <event>  
            - 触发器的触发事件  
            - INSERT  
            - DELETE  
            - UPDATE  
            - UPDATE OF <column_name>

知识点–概念：trigger body 中 IF 语句使用触发事件作为条件

    1.触发器的触发事件可以用在 trigger body 中的 IF 语句中作为判断条件  
        # IF  
            INSERTING THEN ... ;  
            ELSIF UPDATING THEN ... ;  
            ELSIF DELETING THEN ... ;  
            ELSE ... ;  
        END IF ;  
    2.对于 IF UPDATING，可将其拆分为多个 IF UPDATING ('<column_name>')  
        # 此时 IF UPDATING 便不可用

知识点–概念：OLD NEW qualifier，old 限定符和 new 限定符

    1.在 DML FOR EACH ROW 触发器中，在 trigger body 中，在 INSERT 和 UPDATE 和 DELETE 事件中，可以用两个限定符来表示每一次操作的操作行的旧值和新值  
        # :OLD 代表旧值  
        # :NEW 代表新值  
        # 若没有则自动置 null  
    2.两个变量由 oracle 隐式声明，类型为 RECORD，并且以触发器的基表为准  
    3.REFERENCING [OLD AS <old_val_name>] [NEW AS <new_val_name>]  
        # 将 :OLD 改名为 <old_val_name>  
        # 将 :new 改名为 <new_val_name>  
    4.范例  
        # tri_salary 的作用，如果员工的职位不是 AD_VP 或 AD_PRES，则不允许将员工工资调整到 15000 元以上  
            - CREATE OR REPLACE TRIGGER tri_salary  
                BEFORE INSERT OR UPDATE OF salary ON employees  
                REFERENCING OLD AS old_record NEW AS new_record  
                FOR EACH ROW  
                BEGIN  
                    IF NOT ( :old_record.job_id IN ('AD_VP','AD_PRES') ) AND :new_record.salary > 15000 THEN  
                        RAISE_APPLICATION_ERROR( -20001, 'Employee cant earn more than $15000.') ;  
                    END IF ;  
                END tri_salary ;  
                /
