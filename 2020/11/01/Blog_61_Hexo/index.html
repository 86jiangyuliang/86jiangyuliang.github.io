Blog_61_Hexo

知识点：Hexo 介绍

    1. Hexo 默认使用 highlight.js 引擎进行代码块的渲染
    2. Hexo 默认安装了 hexo-renderer-marked 和 hexo-renderer-ejs，因而可以使用 markdown 或 ejs 写文章
    3. Hexo 配置文件支持 yml 和 json 格式
    4. Hexo 文件夹目录
        |-- _config.yml            <== 全局配置文件
        |-- package.json        <== 应用程序的信息。EJS、Stylus、Markdown renderer 已默认安装，您可以自由移除。
        |-- scaffolds            <== 模版文件夹，存放布局的模板文件
        |-- source                <== 资源文件夹，存放用户资源的地方。除 _posts 文件夹之外，开头命名为 _ (下划线)的文件 / 文件夹和隐藏的文件将会被忽略。Markdown 和 HTML 文件会被解析并放到 public 文件夹，而其他文件会被拷贝过去。
        |   |-- _drafts                <== 草稿源文件
        |   |-- _posts                <== 文章源文件
        |-- themes                <== 主题文件夹。Hexo 会根据主题来生成静态页面。
    5. Hexo 组件
        # hexo-server，负责运行 hexo 服务器，单独安装语句 npm install hexo-server --save

知识点：Hexo 命令

    1. 通用选项
        # --safe
            - 安全模式
            - 不会载入插件和脚本
            - 当您在安装新插件遭遇问题时，可以尝试以安全模式重新执行
        # --debug
            - 调试模式
            - 在终端中显示调试信息并记录到 debug.log
            - 当您碰到问题时，可以尝试用调试模式重新执行一次，并提交调试信息到 GitHub
        # --silent
            - 简洁模式
            - 隐藏终端信息
        # --config <config_file>[,...]
            - 自定义配置文件，此时不使用默认的 _config.yml
            - 重复项以后面的文件为主
        # --cwd <path>
            - 自定义工作目录 current working directory
    2. 格式
        # hexo version                                <== 显示 Hexo 版本
        # hexo init <folder>                        <== 新建一个新网站，之后还需要执行 cd <folder>; npm install ;
        # hexo clean                                <== 清除缓存文件 (db.json) 和已生成的静态文件 (public)
        # hexo new [<layout>] [<option>] "<title>"    <== 新建一篇文章
            - <option>
                -p <path>, --path <path>                <== 自定义新文章的路径
                -r, --replace                            <== 如果存在同名文章，将其替换
                -s <slug>, --slug <slug>                <== 文章的 Slug，作为新文章的文件名和发布后的 URL
            - <layout>                                <== 默认三种布局模板，也可以自己创建新的布局
                page
                post
                draft
        # hexo generate [<option>]                    <== 生成生成静态文件，简写为 hexo g
            - <option>
                -d, --deploy                            <== 文件生成后立即部署网站
                -w, --watch                                <== 监视文件变动
                -b, --bail                                <== 生成过程中如果发生任何未处理的异常则抛出异常
                -f, --force                                <== 强制重新生成文件，相当于 hexo clean && hexo generate
                -c, --concurrency                        <== 最大同时生成文件的数量，默认无限制
        # hexo deploy [<option>]                    <== 部署网站
            - <option>
                -g, --generate                            <== 部署之前预先生成静态文件
        # hexo server [<option>]                    <== 启动服务器，默认情况下为 http://localhost:4000/
            - <option>
                -p <port>, --port <port>                <== 重设端口
                -s, --static                            <== 只使用静态文件
                -l, --log                                <== 启动日记记录，使用覆盖记录格式
                -i <address>                            <== 指定地址，可以将服务器定位在内网 ip 上，继而在其他主机上访问
        # hexo publish [<layout>] <filename>        <== 发表草稿
        # hexo render [<option>] <filename>[ ...]    <== 渲染文件
            - <option>
                -o, --output                            <== 设置输出路径
        # hexo migrate <type>                        <== 从其他博客系统迁移内容
        # hexo list <type>                            <== 列出网站资料

知识点：Hexo 配置文件 _config.yml

    1. 网站 site
        # title                <== 网站标题
        # subtitle            <== 网站副标题
        # description        <== 网站描述，主要用于 SEO
            - 告诉搜索引擎一个关于您站点的简单描述
            - 通常建议在其中包含您网站的关键词
        # keywords            <== 网站的关键词，支援多个关键词
        # author            <== 您的名字
        # language            <== 网站使用的语言。对于简体中文用户来说，使用不同的主题可能需要设置成不同的值，请参考你的主题的文档自行设置，常见的有 zh-Hans和 zh-CN。
        # timezone            <== 网站时区
            - Hexo 默认使用您电脑的时区，一般对于中国大陆地区可以使用 Asia/Shanghai。
            - 时区列表 https://en.wikipedia.org/wiki/List_of_tz_database_time_zones
    2. 网址 URL
        # url                    <== 网址，必须以 http:// 或 https:// 开头
        # root                    <== 网站根目录
        # permalink                <== 文章的永久链接格式
            - 永久连接 https://hexo.io/zh-cn/docs/permalinks
        # permalink_defaults    <== 永久链接中各部分的默认值
        # pretty_urls            <== 改写 permalink 的值来美化 URL
            - pretty_urls.trailing_index        <== 是否在永久链接中保留尾部的 index.html，设置为 false 时去除
            - pretty_urls.trailing_html            <== 是否在永久链接中保留尾部的 .html, 设置为 false 时去除 (对尾部的 index.html无效)
    3. 目录 Directory
        # source_dir            <== 资源文件夹，这个文件夹用来存放内容
        # public_dir            <== 公共文件夹，这个文件夹用于存放生成的站点文件
        # tag_dir                <== 标签文件夹
        # archive_dir            <== 归档文件夹
        # category_dir            <== 分类文件夹
        # code_dir                <== Include code 文件夹，source_dir 下的子目录
        # i18n_dir                <== 国际化（i18n）文件夹
        # skip_render            <== 跳过指定文件的渲染。匹配到的文件将会被不做改动地复制到 public 目录中。您可使用 glob 表达式来匹配路径。
            - 样例
                skip_render: "mypage/**/*"                <== 会直接将 'source/mypage/ 下的文件不做改动地输出到 'public' 目录
                skip_render: "_posts/test-post.md"        <== 忽略对 'test-post.md' 的渲染
    4. 文章 Writing
        # new_post_name                <== 新文章的文件名称
            - 支持的参数
                :title                        <== 表示标题，小写、空格将会被替换为短杠
                :year                        <== 建立的年份
                :month                        <== 建立的月份，有前导零
                :i_month                    <== 建立的月份，无前导零
                :day                        <== 建立的日期，有前导零
                :i_day                        <== 建立的日期，无前导零
        # default_layout            <== 预设布局
        # auto_spacing                <== 在中文和英文之间加入空格
        # titlecase                    <== 把标题转换为 title case
        # external_link                <== 在新标签中打开链接
            - external_link.enable            <== 在新标签中打开链接
            - external_link.field            <== 对整个网站（site）生效或仅对文章（post）生效
            - external_link.exclude            <== 需要排除的域名。主域名和子域名如 www 需分别配置
        # filename_case                <== 把文件名称转换为 (1) 小写或 (2) 大写
        # render_drafts                <== 显示草稿
        # post_asset_folder            <== hexo new 时自动启动 Asset 辅助文件夹
        # relative_link                <== 把链接改为与根目录的相对位址
        # future                    <== 显示未来的文章
        # highlight                    <== 代码块的设置，highlight.js 引擎
            - highlight.enable
            - highlight.line_number
            - highlight.auto_detect
            - highlight.tab_replace
            - highlight.wrap
            - highlight.hljs
        # prismjs                    <== 代码块的设置, PrismJS 引擎
    5. 主页 Home page setting
        # index_generator            <== Generate an archive of posts, powered by hexo-generator-index    
            index_generator.path            <== Root path for your blog’s index page
            index_generator.per_page        <== Posts displayed per page.
            index_generator.order_by        <== Posts order. Order by descending date (new to old) by default.
            index_generator.pagination_dir    <== URL format, see Pagination setting below
    6. 分类和标签
        # default_category            <== 默认分类
        # category_map                <== 分类别名
        # tag_map                    <== 标签别名
    7. 日期时间格式
        # date_format                <== 日期格式
        # time_format                <== 时间格式
        # updated_option            <== 当 Front Matter 中没有指定 updated 时 updated 的取值，可选项包括 mtime、date、empty
    8. 分页 Pagination
        # per_page                    <== 每页显示的文章量，0 表示关闭分页功能
        # pagination_dir            <== 分页目录
    9. 扩展 Extensions
        # theme                        <== 当前主题名称，false 表示禁用主题
        # theme_config                <== 主题的配置文件，在这里放置的配置会覆盖主题目录下的 _config.yml 中的配置
        # deploy                    <== 部署部分的设置
        # meta_generator            <== Meta generator 标签，值为 false 时 Hexo 不会在头部插入该标签
    10. 包括或不包括目录和文件 Include/Exclude Files or Folders
        # 列表中的每一项都必须用单引号或双引号包裹起来
        # include 和 exclude 并不适用于 themes/ 目录下的文件。如果需要忽略 themes/ 目录下的部分文件或文件夹，可以使用 ignore 或在文件名之前添加下划线 _
        # ignore 适用于所有目录
        # include                    <== Hexo 默认会忽略隐藏文件和文件夹（包括名称以下划线和 . 开头的文件和文件夹，Hexo 的 _posts 和 _data 等目录除外）。通过设置此字段将使 Hexo 处理他们并将它们复制到 source 目录下。
            - 样例
                include:
                  - ".nojekyll"            
                  - "css/_typing.css"        <== 包括 'source/css/_typing.css'
                  - "_css/*"                <== 包括 'source/_css/' 中的任何文件，但不包括子目录及其其中的文件
                  - "_css/**/*"                <== 包含 'source/_css/' 中的任何文件和子目录下的任何文件
        # exclude                    <== Hexo 会忽略这些文件和目录
            - 不要用 exclude 来忽略 'source/_posts/' 中的文件。你应该使用 'skip_render'，或者在要忽略的文件的文件名之前加一个下划线 '_'
            - 样例
                exclude:
                    - "js/test.js"            <== 不包括 'source/js/test.js'
                    - "js/*"                <== 不包括 'source/js/' 中的文件、但包括子目录下的所有目录和文件
                    - "js/**/*"                <== 不包括 'source/js/' 中的文件和子目录下的任何文件
                    - "js/test*"            <== 不包括 'source/js/' 目录下的所有文件名以 'test' 开头的文件，但包括其它文件和子目录下的单文件
                    - "js/**/test*"            <== 不包括 'source/js/' 及其子目录中任何以 'test' 开头的文件
        # ignore                    <== Ignore files/folders，include and exclude options only apply to the source/ folder, whereas ignore option applies to all folders.
            - 样例
                ignore:
                    - "**/foo"                <== Ignore any folder named 'foo'.
                    - "**/themes/*/foo"        <== Ignore 'foo' folder in 'themes/' only.
                    - "**/themes/**/foo"    <== Same as above, but applies to every subfolders of 'themes/'.

知识点：Hexo Front-matter

    1. 介绍
        # JSON Front-matter
            - 除了 YAML 外，你也可以使用 JSON 来编写 Front-matter，只要将 --- 代换成 ;;; 即可
            - 样例
                "title": "Hello World",
                "date": "2013/7/13 20:46:25"
                ;;;    
    2. 配置项
        # layout
        # title
            - 文章标题
        # date
            - 文章的创建时间
        # updated
            - 文字的更新时间
        # comments
            - 开启文章的评论功能
            - 默认值为 ture
        # tags
            - 文章标签
            - 不支持层次性，支持并列
        # categories
            - 文章分类
            - 支持层次性，不支持并列
            - 样例
                categories:                        <== 这篇文章同时包括三个分类，PlayStation、Games、Life
                - [Diary, PlayStation]            <== PlayStation 是父分类 Diary 的子分类
                - [Diary, Games]                <== Games 是父分类 Diary 的子分类
                - [Life]                        <== Life 是一个没有子分类的分类
        # permalink
            - 覆盖文章网址    
        # excerpt
            - 文章摘要
            - Page excerpt in plain text. Use this plugin to format the text
        # disableNunjucks
            - Disable rendering of Nunjucks tag {{ }}/{% %} and tag plugins when enabled

知识点：Hexo 标签插件 Tag Plugins

    1. 介绍
        # 标签插件和 Front-matter 中的标签不同，它们是用于在文章中快速插入特定内容的插件。
        # 虽然你可以使用任何格式书写你的文章，但是标签插件永远可用，且语法也都是一致的。
        # Tag plugins should not be wrapped inside Markdown syntax, e.g. []() is not supported.
        # Tag plugins 应该单独使用，不能被包裹在 markdown 语法中
            - 本知识点有部分内容无法在网页页面中显示，具体查看源码
