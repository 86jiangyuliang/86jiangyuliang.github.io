2.4.1 pssh 指令

知识点–指令：pssh

    1. 介绍 Description
        # pssh，可以并发 ssh 并在远程主机执行相应的命令
    2. 常用用法 Usage
        # pssh -i -H "host1 host2" echo "hello, world"                                <== 登录到两台主机，让其打印出 hello world
        # pssh -i -h hosts.txt echo "hello, world"                                    <== 通过文件指定远程主机，打印出 hello world
        # pssh -i -h hosts.txt -o output echo "hello, world"                        <== 将输出保存到 output 目录中
        # pssh -i -h hosts.txt -A echo "hello world"                                <== 指定 prompt 以输入密码
        # pssh -i -h hosts.txt -O StrictHostKeyChecking=no echo "hello world"        <== 在建立连结时直接添加远程主机的指纹到 known_hosts 文件中
        # pssh -H host1 -I < script                                                    <== 远程运行本地脚本
    3. 指令格式 Synopsis
        # pssh [<options>] <command> [...]                            <== 如果指定多个命令，则除了最后一个命令，前面的每一个命令中必须包含分号 ;
        # pssh -I [<command> [...] ]
    4. 选项 Options
        # -H [<user>@]<host>[:<port>]
          --host [<user>@]<host>[:<port>]
          -H "[<user>@]<host>[:<port>] [ ... ]"
          --host "[<user>@]<host>[:<port>] [ ... ]"        <== 直接在命令行中指定主机，可以多次使用
        # -h <host_file>， --hosts <host_file>            <== 从文件中读取主机，格式为 [<user>@]<host>[:<port>]，可以包含空行和 # 注释，可以多次使用
        # -l <user> ， --user <user>                    <== 指定登录的用户名
        # -p <parallelism>， --par <parallelism>        <== 指定每次并行的数量。如果 -h 指定主机并且数量较多时，适合使用此选项
        # -t <timeout>, --timeout <timeout>                <== 指定超时的时间，0 表示 pssh 不会进行超时断连
        # -o <outdir>, --outdir <outdir>                <== 将 STDOUT 保存到指定的目录中
        # -e <errdir>, --errdir <errdir>                <== 将 STDERR 保存到指定目录
        # -x <args>, --extra-args <args>                <== 发送额外的的 ssh 命令行参数，<args> 需要放在引号中并且可以包含多个项，可以多次指定
        # -X <arg>, --extra-arg <arg>                    <== 只发送一个 ssh 命令行参数，需要放在引号中，可以多次指定
        # -O <options>, --options <options>                <== 直接在 pssh 上使用 ssh 的 -o 选项，可以多次指定
        # -A, --askpass                                    <== 要求 pssh 提供 prompt 以输入密码。
        # -i, --inline                                    <== 直接显示出 STDOUT 和 STDERR
        # --inline-stdout                                <== 仅显示出 STDOUT
        # -v --verbose                                    <== 繁冗模式
        # -I, --send-input                                <== 读取输入，并发送到每个 ssh 进程，可以实现在远程主机运行本地脚本，如果是交互模式，则结束时需要输入 Ctrl + D 表示 eof

2.5.1 sshpass 指令

知识点–指令：sshpass

    1. 介绍 Description
        # sshpass，实现非交互方式的 ssh 密码登录
        # sshpass 会运行一个 tty，并在此 tty 中运行 ssh，并将密码传递给 ssh
    2. 常用用法 Usage
        # 在命令行中指定密码
            * sshpass -p <password> ssh <host>
        # 在 rsync 中调用 sshpass，以建立 ssh 通道
            * rsync --rsh='sshpass -p 12345 ssh' <host>:<path>
    3. 指令格式 Synopsis
        # sshpass [<password_option>] [<other_options>] <command> <arguments>
        # <command> 是实际在 tty 中运行的 ssh 指令，可以是 ssh、scp 等
    4. 密码选项 PASSWORD OPTIONS
        # 密码选项用来指定 sshpass 获取密码的方式，如果未指定则表示从 STDIN/键盘输入来获取
        # -p <password>        <== 直接将密码放在命令行上
        # -f <filename>        <== 密码放在指定文件的第一行。
        # -d <number>        <== 密码通过 file descriptor 来获得
        # -e                <== 密码通过环境变量 SSHPASS 来获得
    5. 其他选项 OTHER OTPIONS
        # -P <prompt>        <== 设置密码的 prompt
        # -v                <== 繁冗模式---

知识点–概念：软件包管理器、软件包线上升级机制、rpm、cpkg、yum、apt

    1. 软件包管理器
        # 用户使用厂商的系统 distribution
        # distribution 厂商事先在固定的硬件平台和系统平台上编译好相关的软件
        # distribution 厂商再将编译好的 binary program 打包提供给用户使用
        # 用户直接使用 distribution 厂商提供的软件包来安装软件，不需要编译
        # 软件包在安装之前，会自行检测用户的系统环境和依赖软件，符合条件就会进行安装
        # 由于用户使用的就是厂商的系统，系统与厂商编译时的环境相同，因此厂商编译的 binary program 用户可以直接使用
        # 用户在安装厂商给出的软件时，再将安装产生的信息记录下来，形成一个数据库
        # 这样用户在卸载软件，升级软件时，根据数据库的内容，可以简单方便的进行
        # 这种方式就叫做软件包管理器
    2. 关于 megacli 没法 yum 的原因
        # 个人认为就是因为 distribution 厂商没法确定用户的硬件环境，所以就不做 yum 了
    3. 软件包线上升级机制
        # 主要为了解决软件的依赖问题
        # 在安装软件之间，如果检测到需要安装其他的依赖的软件，就会先从线上下载相关的软件安装，再安装本来的软件
    4. linux 界的两大主流软件包管理器，rpm，dpck
        # redhat 使用 rpm，线上升级用 yum
        # debian 使用 dpck，线上升级用 apt
        # SuSE 线上升级用 YOU

知识点–概念：RPM，SRPM

    1. RPM
        # RedHat Package Manager
        # 是以一种以数据库记录的方式来将所需要的软件安装到 Linux 系统的一套管理机制
            * 厂商将软件编译好后，打包成 rpm 包
            * 用户使用 rpm 指令来直接解压缩 rpm 包
            * rpm 指令会根据 rpm 包里的内容，和用户主机里的 rpm 数据库进行对比，检查软件的依赖性，检查通过则会进行安装
            * 安装时，会将安装信息记录到 rpm 数据库中
        # 优点
            * 便于安装，不需要编译
            * 安装前会检查依赖性和系统资源，防止错误安装
            * 便于了解软件的信息
            * 建立软件信息数据库，便于查询，升级，卸载
        # 缺点
            * 软件安装的环境必须与打包时的环境需求一致或相当
                - 通常不同的 distribution 的 RPM 包不能通用
                - 甚至相同 distribution 的不同版本之间也无法互通
            * 需要满足软件的依赖关系
            * 卸载时需要特别小心，最底层的软件不可先卸载，否则可能造成整个系统的问题
    2. SRPM
        # Source RPM
        # 软件的源码包，一般副文件名都是 .src.rpm
            * redhat 在放出 rpm 包时，会一起释放出软件的原始码
            * 这样用户就可以根据自己的系统的情况来进行编译软件
        # SRPM 包里的内容
            * 软件的源码
            * 软件的依赖性说明
            * 所有 RPM 文件所提供的资料
            * 编译所需要的参数设定文件，也就是 configure 与 makefile
        # SRPM 包的用法
            * 先将该软件以 RPM 管理的方式编译，此时 SRPM 会被编译成为 RPM 文件
            * 然后将编译完成的 RPM 文件安装到 Linux 系统当中
            * 与原版 RPM 包的区别，用户可以根据自己的系统环境对生成的 RPM 包进行修改，产生适合自己系统环境的 RPM 包
        # Centos 和 SRPM
            * redhat 释放出 SRPM 后，社区中的人将这些 SRPM 收集起来，重新编译成为所需要的软件，再重复释出成为 CentOS

知识点–概念：rpm 包的一些知识

    1. rpm 包的命名规则
        # <program>-<version>-<pack>.<el_version>-<hardware>.rpm
            * pack 打包次数，多次打包的原因一般是修复一些小 bug
            * el version，就是针对的 distribution 的版本，例如 centos7 就需要安装 el7 的软件
    2. rpm 包支持的硬件平台
        # rpm 包主要支持的硬件平台有，i386，i586，i686，x86_64，noarch
        # noarch
            * 没有任何硬件上的限制
            * 一般这种类型的 RPM 文件里面没有 binary program，通常都是 shell script
        # i386，i586，i686
            * i 指的是 Intel 相容的 CPU
            * i386，针对 386 CPU 平台所设计，几乎适用于所有的 x86 平台
        # x86_64
            * 针对 64 位的 CPU 进行设计
        # 由于硬件平台都支持向下兼容，所以旧版本的软件可以安装的高级的硬件上
            * 但是如果强调效能的话，还是最好选择搭配的 RPM 包
    3. rpm 包的数字签名
        # 数字签名的作用
            * 安装 rpm 包时，会校验原厂公钥文件
                - rpm 指令安装 rpm 包时，会先读取 RPM 文件的签名信息，与本机系统内安装的原厂数字签名信息进行对比
                - 若签名相同则安装
                - 若签名不同则给予警告并且停止安装
        # CentOS 使用的数字签名系统为 GNU 计划的 GnuPG(GNU Privacy Guard，GPG)
        # 在本机上安装数字签名
            * 使用 rpm --import，导入镜像中自带的数字签名文件 GPG-KEY*
            * 数字签名文件可以当作一个普通的 rpm 包查看，通过指令 rpm -qa | grep pubkey
            * 不同的 distribution 的数字签名放在不同的地方，一般文件名中都含有 GPG-KEY
            * Centos 的数字签名文件放在 /etc/pki/rpm-gpg 下
            * 实际一般在安装系统时，已经自动导入了数字签名文件

7.1.4 Bash 执行程序

知识点–概念：Bash 执行程序

    1. 当执行命令时，shell 将等待进程的结束， 这时shell 处千非活动状态，称为休眠(sleep)状态。
    2. 程序执行完毕， 它将其退出状态传递给shell, 这样shell就返回活动状态（被唤醒）， 显示提示符， 等待下一条命令的输入。
    3. shell不处理进程的参数由千shell不处理命令行上的参数，只是将它们传递给调用的程序，因此shell并不知道特定选项和参数是否对给定程序有效。

7.1.4 标准输入和标准输出

知识点–概念：标准输入和标准输出

    1. 标准输出(standard output)指程序输出信息（如文本）的地方。程序从来都不“ 知道“ 自已发送到标准输出的信息究竟要送往何处
    2. 标准输入(standard input)是程序获取信息的地方。默认情况下，shell从键盘定向标准输入。与标准输出一样，程序也从不“ 知道“ 信息的来源。
    3. 按照约定， 进程希望它的调用程序（常常是shell)已经设置了标准输入、标准输出和标准错误， 这样进程就可以立即使用它们。
        # 被调用的进程不必知道哪些文件或设备已连接到标准输入、标准输出或标准错误上．
        # 但是， 进程可查询内核以荻取连接到标准输入、标准输出或标准错误的设备信息。
        # 例如，ls实用程序在把输出显示在屏幕上时， 会显示在多列上， 但如果输出被重定向到一个文件或另一个程序中，ls就生成单列输出。
        # ls实用程序使用isatty()系统调用来确定输出是否显示在屏幕上。

    for <name> [ [ in [ <word_list> ... ] ] ; ] do <list> ; done

            The list of words following in is expanded, generating a list of items.
            The variable name is set to each element of this list in turn, and list is executed each time.
            If the in word is omitted, the for command executes list once for each positional parameter that is set (see PARAMETERS below).
            The return status is the exit status of the last command that executes.
            If the expansion of the items following in results in an empty list, no commands are executed, and the return status is 0.

    for (( <expr_1> ; <expr_2> ; <expr_3> )) ; do <list> ; done

            First, the arithmetic expression expr1 is evaluated according to the rules described below under ARITHMETIC EVALUATION.
            The arithmetic expression expr2 is then evaluated repeatedly until it evaluates to zero.
            Each time expr2 evaluates to a non-zero value, list is executed and the arithmetic expression expr3 is evaluated.
            If any expression is omitted, it behaves as if it evaluates to 1.
            The return value is the exit status of the last command in list that is executed, or false if any of the expressions is invalid.

    select <name> [ in <word> ] ; do <list> ; done

            The list of words following in is expanded, generating a list of items.
            The set of expanded words is printed on the standard error, each preceded by a number.
            If the in word is omitted, the positional parameters are printed (see PARAMETERS below).
            The PS3 prompt is then displayed and a line read from the standard input.
            If the line consists of a number corresponding to one of the displayed words, then the value of name is set to that word.
            If the line is empty, the words and prompt are displayed again.
            If EOF is read, the command completes.
            Any other value read causes name to be set to null.
            The line read is saved in the variable REPLY.
            The list is executed after each selection until a break command is executed.
            The exit status of select is the exit status of the last command executed in list, or zero if no commands were executed.

    case <word> in [ [(] <pattern> [ | <pattern> ] ... ) <list> ;; ] ... esac

            A case command first expands word, and tries to match it against each pattern in turn, using the same matching rules as for pathname expansion (see Pathname Expansion below).
            The word is expanded using tilde expansion, parameter and variable expansion, arithmetic substitution, command substitution, process substitution and quote removal.
            Each pattern examined is expanded using tilde expansion, parameter and variable expansion, arithmetic substitution, command substitution, and process substitution.
            If the shell option nocasematch is enabled, the match is performed without regard to the case of alphabetic characters.
            When a match is found, the corresponding list is executed.
            If the ;; operator is used, no subsequent matches are attempted after the first pattern match.
            Using ;& in place of ;; causes execution to continue with the list associated with the next set of patterns.
            Using ;;& in place of ;; causes the shell to test the next pattern list in the statement, if any, and execute any associated list on a successful match.
            The exit status is zero if no pattern matches.
            Otherwise, it is the exit status of the last command executed in list.

    if <list>; then <list>; [ elif <list>; then <list>; ] ... [ else <list>; ] fi

            The if list is executed.
            If its exit status is zero, the then list is executed.
            Otherwise, each elif list is executed in turn, and if its exit status is zero, the corresponding then list is executed and the command completes.
            Otherwise, the else list is executed, if present.
            The exit status is the exit status of the last command executed, or zero if no condition tested true.

    while <list1>; do <list2>; done
    until <list1>; do <list2>; done

            The while command continuously executes the list list-2 as long as the last command in the list list-1 returns an exit status of zero.
            The until command is identical to the while command, except that the test is negated; list-2 is executed as long as the last command in list-1 returns a non-zero exit status.
            The exit status of the while and until commands is the exit status of the last command executed in list-2, or zero if none was executed.

知识点–概念：shell scripts 的编写注意事项

2.指令,选项与参数间的多个空白都会被识别成一个空格
3.空白行，^$，将被忽略掉
4.制表符 [tab] 键，会被视为空白键
4.如果读取到一个 Enter 符号 CR，就尝试开始执行该行或该串命令
5.至于如果一行的内容太多，则可以使用跳脱 Enter 的方式来延伸至下一行
6.# 可做为注解，在除了 shebang 以外的其他行，任何加在 # 后面的资料将全部被视为注解文字而被忽略
7.当在脚本内进行重定向到文件时，如果只写文件名或使用相对路径，新文件会被建立在执行命令的目录
8.重定向到文件，使用变量做文件名：echo “ Hello World “ >> ${filename} ，filename 必须是已经提前定义好文件名的变量
知识点–概念：shell script 的执行方式

#变量 PATH 功能，将 shell.sh 放在 PATH 指定的目录内，例如： /bin/
2.以 bash 程序来执行，最重要的是 shell.sh，只要有 r 权限就行，不需要 x 权限
#bash shell.sh 或 sh shell.sh
知识点–概念：shebang，#! ，它指明了执行这个脚本文件的解释程序
1. 注意，#!之后的解释程序，需要写其绝对路径，如，#!/bin/bash

第二次循环时，$var 的内容为 con2
第三次循环时，$var 的内容为 con3

程序段

第一次循环时，$var 的内容为 con1
第二次循环时，$var 的内容为 con2
第三次循环时，$var 的内容为 con3

9.3 awk

知识点–指令：awk，以列为单位进行文本处理

    1.用法
        # awk '条件类型1{动作1} 条件类型2{动作2} ...' filename
        # awk 主要是处理'每一列的内容'，默认的栏位的分隔符号为空格键或制表符 [tab]
        # awk 中后续的所有动作都要用单引号 ' 括住，并且单引号是 awk 的固定用法，所以如果动作里面需要用到引号就只能是双引号
            并且双引号还不能嵌套，只能从前向后依次成对出现
        # 如果动作前没有加条件，则代表将对资料中所有内容进行此动作
    2.内建变量，awk 中有内建变量
        # 内建变量，除了列号，其他的如 NR，使用时不需要加 $ 号
        # $<N>      -- 每一列的变量名称，$1 就是第一列，$2 就是第二列
        # NF，内容为文件总共有多少列
        # NR，代指目前 awk 所处理的是第几'行'资料
        # FS        -- 指定列分隔符，默认是空白键
    3.awk 的逻辑运算字符
        # > ，大于
        # < ，小于
        # >=    ，大于或等于
        # <=    ，小于或等于
        # ==    ，等于
        # !=    不等于
    4.awk 的条件判断，也可以使用 if 语句
    5.awk 的条件变量，BEGIN，设定动作从第一行就开始执行，如果不加，在读取第一行的时候，那些变量 $1, $2 默认还是以空白键为分隔
    6.awk 的动作，如果一个动作内包含多个指令，则两个指令之间可以用分号间隔，或者跳脱回车也可以
    7.awk 用 printf 动作来列印，默认是不会断行的，所以 printf 最后要加上 \n 来断行
    7.例子
        # last | \
            > awk '{print $1 "\t" $3}'
        # cat /etc/passwd | \
            > awk 'BEGIN {FS=":"} $3 < 10 {print $1 "\t " $3}'
            - 重点在 BEGIN ，第一个动作是专门给 BEGIN 用的，将分隔符设定为冒号
        # cat pay.txt | \
            > awk 'NR==1{printf "%10s %10s %10s %10s %10s\n",$1,$2,$3,$4,"Total" }
            > NR>=2{total = $2 + $3 + $4 ; printf "%10s %10d %10d %10d %10.2f\n", $1, $2, $3, $4, total}'，重点在那个分号上，用分号来间隔动作
        # cat pay.txt | \ 
            > awk '{if(NR==1) printf "%10s %10s %10s %10s %10s\n",$1,$2,$3,$4,"Total "} 
            > NR>=2{total = $2 + $3 + $4 
            > printf "%10s %10d %10d %10d %10.2f\n", $1, $2, $3, $4, total}'
            - 与上一个例子相比，最大的不同就是判断改成用 if 语句来判断了

13.3.4 特殊权限

知识点–概念：文件和目录的特殊权限

    1.特殊权限有三个
        # SUID
        # SGID
        # SBIT
    2.文件的特殊权限设定后，若正常可使用，应该是小写字母
        # 如果出现大写字母，则代表权限条件不足
        # 例如设置了 SUID，但执行者没有对程序的 x 权限，此时 SUID 就会是大写的 S
        # 这种情况下，虽然设置了特殊权限，但特殊权限其实不起作用

知识点–概念：SUID

    1.Set UID
        # 仅对二进制程序文件有效，对 shell script 没用
        # 在二进制程序文件的拥有者的 x 权限出现 s 标志，称为 Set UID
        # 作用
            - 非程序的拥有者执行该程序时，能获得该程序拥有者的权限
            - 程序的运行时间结束后，执行者的权限又恢复为原来的状态
            - 执行者要拥有程序的 x 权限
    2.范例
        # 普通用户利用 passwd 来修改自己的密码
            - 正常情况下，普通用户无权修改 /etc/passwd、/etc/shadow 这几个文件
            - 但 passwd 设置了 SUID
            - 那么普通用户使用 passwd 指令时，就可以修改这几个文件了
            - 当 passwd 指令执行结束，普通用户的权限又恢复到原来的状态了

知识点–概念：SGID

    1.Set GID
        # 可以针对文件或目录来设定
        # 在文件组拥有者的 x 权限出现 s 标志，称为 Set GID
        # 程序文件 set UID 作用
            - 执行者在执行的过程中将会获得该程序组群的支持
            - 执行者要拥有该程序的 x 权限
        # 目录 set UID 作用
            - 用户若对于此目录具有 r 与 x 的权限时，该用户能够进入此目录
            - 用户在此目录下的有效组群 effective group将会变成该目录的组群
            - 当用户离开此目录时，其有效组群又恢复正常
    2.用途
        # 主要用在软件开发，让开发人员不用反复切换自己的主要组
        # 普通情况
            - 用户在某目录中建立的文件，必须切换适当的组群身份
            - 否则其他相关人员没法查看和修改，因为组群不同，要么就是所有人都能看和修改
        # set gid
            - 则用户在此文件下新建的文件，会自动的设置为目录的组拥有者

知识点–概念：SBIT，Sticky Bit

    1.other 权限的 x 位置上出现 t
    2.只针对目录有效，对于文件已经没有效果了
    3.若目录设定了 SBIT
        # 当用户对于此目录具有 w，x 权限，亦即具有新建文件或修改文件的能力时
        # 用户在该目录下建立文件或目录时，仅有自己与 root 才有权力删除该文件

知识点–操作：设置 SUID/SGID/SBIT，使用 chmod

    1.chmod 4755 filename
    2.chmod g+s，o+t filename
    3.数字表示特殊权限
        # 4 为 SUID
        # 2 为 SGID
        # 1 为 SBIT

知识点–概念：MBR Master Boot Record

15.2.1 索引式文件系统

知识点–概念：索引式文件系统，superblock，inode，block

    1.索引式文件系统
        # 索引式文件系统的结构，图片 07-01-01
        # FAT 式文件系统的结构，图片 07-01-02
    2.superblock
    3.block
        # 每个 block 都有编号，大小为 4B 
        # 大小有 1K、2K、4K 等，通常使用 4K
        # 原则上，block 的大小与数量在格式化后便无法改变
        # 一个 block 只能放一个文件的内容
    4.inode
        # 一个文件与一个 inode 一一对应
        # 系统读取文件时，先找到 inode，根据 inode 分析权限，符合权限才进一步读取 block 的内容
        # inode 里并不记录文件名，文件名被保存在目录的 block 中
        # inode 记录的内容
            - 权限 rwx
            - 拥有者与拥有者组
            - 大小
            - mtime、atime、ctime
            - SUID、SGID、SBIT
            - block id
        # inode 记录 block 号码的方式
            - inode 记录一个 block 号码要花掉 4B，故 ext2 最多支持 2^32 = 4G 个 block 
            - inode 记录 block 号码的区域定义为 12 个直接，1 个间接，1 个双间接，1 个三间接
            - 12 个直接指向 12 个存放文件数据的 block
            - 1 个间接指向 1 个 block，最后指向存放文件数据的 block
            - 1 个双间接指向 1 个双间接 block，再指向 block，最后指向存放文件数据的 block
            - 1 个三间接指向 1 个三间接 block，再指向 block，再指向 block 号码，最后指向存放文件数据的 block
            - 例如，block 为 1KB，则一个文件最大为 (12 + (1024/4) + (1024/4)^2 + (1024/4)^3)*1KB = 16GB
    5.目录的 inode 和 block

        # block 记录所有文件的文件名和 inode 列表
        # 一个目录下的所有文件均不能重名
        # 读取文件的流程
            - 目录的 inode
            - 目录的 block
            - 文件的 inode
            - 文件的 block

知识点–指令：mkfs.xfs

    1.介绍
    2.格式

    3.选项

            agsize=数值 ，每个 AG 设定为多少容量的意思，通常 agcount/agsize 只选一个设定即可

            sw=数值     ，当有 RAID 时，用于储存资料的硬盘数量，须扣除备份碟与备用碟，如 raid5 要减一
            sunit=数值  ，swidth，与 su 相当，不过单位使用的是几个 sector（512bytes大小）的意思

            internal=[0|1]  ，log 设备是否为内建，默认为 1 内建，0 为使用外部设备，如果要用外部设备，要再加上 logdev 参数
            logdev=device   ，log 设备放在为后面接的那个设备上，需设定 internal=0 才可
            size=数值     ，指定这块登录区的容量，通常最小得要有 512 个 block，大约 2M 以上才行
        #-L，设定这个文件系统的标头名称 Label name 
        #-r，指定 realtime section 的相关设定值，常见的有
            extsize=数值，就是那个重要的 extent 数值，一般不须设定，但有 RAID 时，最好设定与 swidth 的数值相同较佳，最小为 4K 最大为 1G

16.1 LVM

16.1.1 LVM

知识点–概念：LVM，Logical Volume Manager

    1.LVM 结构
        # PE，Physical Extent，LVM 最小储存区，由固定数量的连续 block 组成
        # PV，Physical Volume，LVM 的最底层，将一个 partition 或一个 Disk 创建为一个 PV
        # VG，Volume Group，将多个 PV 整合成一个 VG
        # LV，Logical Volume，从 VG 中切割出 LV，设备名 /dev/<vgname>/<lvname>
        # 在 LV 上建立文件系统
    2.LVM 的优点
        # 弹性调整文件系统的大小
            - 在 LV 上建立文件系统
            - 调整 LV 大小通过增减 PE 实现
            - 调整 LV 大小可以在线进行，不需要 umount
            - EXT 既能扩大，也能缩小
            - XFS 只能扩大
            - 扩大文件系统，是采用增加 block group 的方式进行的，即并不是重新格式化，而只是对新增的部分进行格式化
    3.LVM2
        # LVM 的最新版为 LVM2，可以 yum 安装
        # 32-bit 系统，一个 LV 最多能含有 65534 个 PE，LVM1 格式
        # CentOS 6.x 以后，由于直接使用 LVM2，以及系统转为 64 位，因此这个限制已经不存在了

知识点–概念：LVM 的两种硬盘使用的机制

    1.线性模式 linear
        # LVM 的默认模式
        # 线性模式的工作方式
            - 按顺序依次使用 PV
            - 一个 PV 的空间用完后，才使用另一个 PV
        # 因为使用 LVM 的目的主要是实现一个可以弹性调整空间的文件系统，而不是建立一个效能为主的硬盘，效能可以使用 RAID 来做到
    2.交错模式 triped
        # 有点像 RAID 0，理论上，读写的效能会比较好，但文件的安全性会差很多
        # 交错模式的工作方式
            - 同时向所有的 PV 写入
