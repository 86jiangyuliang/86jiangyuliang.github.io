Chapter_01_笔记格式

知识点：笔记 source 子文件规则

    1. 所有的 source 子文件全部为 txt 格式
    2. 每个 source 子文件，最后加几个空行
        # 空行不能包含空白字符
    3. 章标题 source 子文件，文件名遵守格式 Chapter_NN_<chapter_name>.txt
    4. 章标题 source 子文件，第一行应为 ---，表示分隔线
    5. 每次编辑结束前，将所有 tab 需转换为空格，需使用新版本的 notepad
    6. 合并命令
        # for %f in (*.txt) do type "%f" >> Blog_00_Miscellaneous.md

知识点：笔记目录规则

    1. 加粗
        # 目录加粗表示已开始整理
        # 目录未加粗表示还没有开始整理

知识点：笔记标题规则

    1. 章标题
        # 标题中不能带空格
        # 标题遵守格式 Chapter_NN_<chapter_name>
        # 章标题使用一级标题，即一个 # 和四个空格
    2. 节标题
        # 节号与节名之间需加一个空格
        # 节标题使用二级标题，即一个 ## 和三个空格
    3. 目标题
        # 目号与目名之间需加一个空格
        # 目标题使用三级标题，即一个 ### 和两个空格
    4. 知识点标题
        # 知识点标题使用四级标题，即一个 #### 和一个空格

知识点：笔记正文规则

    1. 每一个文本行结尾必须为两个空格
    2. 正文分为四层
        # 第一层，标号为 N.
        # 第二层，标号为 #
        # 第三层，标号为 *
        # 第四层，标号为 -
        # 每一层的标号和正文之间，加一个空格
    3. 正文语法  
        # 中括号 []，表示可选内容  
        # 尖括号 <>，表示手动输入内容  
        # 箭头 <== ，表示行级注释  
    4. 表格
        # 使用 <p> </p> 实现表格内换行
    5. 符号
        # 知识点正文允许使用的符号
            - ，
            - 、

知识点：Blog_01_Linux 的笔记规则

    1. 内容分类
        # 章，取决于功能
        # 节，取决于包
            - 具有近似功能的多个包，可以放在同一节中
            - 包的介绍直接放在节中
        # 目，取决于具体文件，如指令、配置文件
        # 特别说明
            - 两个基础巨型包，coreutils 和 util-linux，其中的内容整合到相应的功能中，不组合到一起，仅在知识点中标注来源包即可
            - Bash 的内置命令，内容整合到相应的功能章节，不组合到一起，仅在知识点中标注来源于 Bash build-in
    2. 指令知识点的格式
        # 知识点标题中需要对指令进行简略描述
        # 1 为介绍 Description
        # 2 为常用用法 Usage
        # 3 为指令格式 Synopsis
            - 如果有多个指令格式，则每种指令格式占用一行
        # 4 为选项 Options
            - 选项的功能说明，全部使用行级注释
        # 5+ 为分类选项
            - 在作为选项过多时，按照功能进行的对选项的分类
        # N-1 为输出 Output
            - 倒数第二项可以作为输出
        # N 为范例 Example
            - 只有最后一项可以作为范例
            - 范例必须加行级注释，哪怕是废话
        # 行级注释
            - 子层的行级注释要比父层的行级注释多两个 tab
            - 上下文的行级注释，相同的层级的尽量拥有相同的 tab

知识点：markdown 块级元素

    1.空行  
        # 不包含任何字符的行  
        # 只包含空白字符的行  
        # 空行被视为段落的间隔  
        # 或连续的多个空行被解析为一个空行  
    2.换行  
        # 以两个或两个以上的空格结尾，再输入回车符  
    3.水平线  
        # 在一行里只放三个或更多个 -、*、_，就会得到一个水平线，一行中的字符必须相同  
    4.atx 风格标题  
        # # <title_level_1>  
        # ## <title_level_2>  
        # ### <title_level_3>  
        # #### <title_level_4>  
        # ##### <title_level_5>  
        # ###### <title_level_6>  
        # atx 风格标题也可以添加结尾 #，仅为了美观，结尾 # 的数量多少不影响标题的级别  
    5.setext 风格标题  
        # level_1 级标题，下一行仅包含 = 符号，一个即可  
        # level_2 级标题，下一行仅包含 - 符号，至少一个即可，并创建水平线  
            - 注意如果水平线是由 * 或 _ 字符创建，则不会将上一行识别为标题  
            - 注意若 - 被识别为无序列表的开头，则不会将上一行识别为标题  
        # setext 风格标题只支持到 level_2 级

知识点：行级元素

    1.字体格式  
        # *<text>*          <-- 斜体  
        # _<text>_          <-- 斜体  
        # **<text>**        <-- 加粗  
        # __<text>__        <-- 加粗  
        # ~~<text>~~        <-- 删除线，注意 ~ 和 <text> 之间不能有空格  
        # <u><text></u>     <-- 下划线

知识点：markdown 引用块

    1. 引用块  
        # 每一行在行首添加 >  
        # 或仅在第一行的行首添加 >，其他行省略 >  
        # 添加多个 > 符号，进行嵌套饮用块  
        # 引用块中的 Markdown 元素会被解析  
        # 引用快遇到空行结束

知识点：markdown 代码和代码块

    1. 代码块  
        # 每一行缩进四个 space 或一个 TAB  
        # 代码块中的字符会直接打印，不解析  
        # 代码块在遇到没有缩进的一行自动结束  
    2. 行内代码  
        # 使用一个或多个连续的反引号包住  
        # 要在行内代码中添加反引号，则起始反引号和结束反引号使用多个连续的反引号，并使用空格间隔，则行内代码中数量较少的反引号会被解析为普通字符  
    3. 代码高亮  
        ```<编程语言>  
            <codes>  
        ```

知识点：markdown 列表

    1. 无序列表  
        # - <content>  
        # + <content>  
        # * <content>  
        # 三种符号可以混用  
    2. 有序列表  
        # <N>. <content>  
        # 有序列表的 <N> 可以随便写  
            - 第一个 <N> 视为有序列表的开始序号  
            - 之后的 <N> 会自动按照在有序列表中的位置进行进位  
        # 有序列表可以嵌套有序列表，子层在父层的基础上，开头加两个 space  
        # 有序列表可以嵌套无序列表，子层在父层的基础上，开头加四个 space  
            - 子层的五序列表若开头仅加了两个 space，则与父层的有序列表平级  
    3. 列表嵌套列表  
        # 子层在父层的基础上，开头加两个 space 或一个 TAB  
        # 无序列表可以嵌套无序列表或有序列表  
        # 有序列表可以嵌套无序列表或有序列表  
    4. 列表嵌套其他块  
        # 列表嵌套代码块，代码块相对于父层，开头加两个 TAB  
        # 列表嵌套饮用可，引用块相对于父层，开头加一个 TAB 和一个 >  
    5. 列表的间隔  
        # 标题可以间隔不同的列表块

知识点：Hexo 标签插件 Tag Plugins

    0. 注意
        # 实际上的符号为 { 和 % 连在一起作为开始，中间不加 \ 也不加空格，以 % 和 } 连在一起作为结尾，中间不加 \ 也不加空格
        # 为了避免对 Blog 页面产生影响，添加了 \ 作为跳脱
    1. 介绍
        # 标签插件和 Front-matter 中的标签不同，它们是用于在文章中快速插入特定内容的插件。
        # 虽然你可以使用任何格式书写你的文章，但是标签插件永远可用，且语法也都是一致的。
        # Tag plugins should not be wrapped inside Markdown syntax, e.g. []({% post_path lorem-ipsum %}) is not supported.
        # Tag plugins 应该单独使用，不能被包裹在 markdown 语法中
            - 本知识点有部分内容无法在网页页面中显示，具体查看源码
    2. 引用块 Block Quote
        # 在文章中插入引言，可包含作者、来源和标题。
        # 别名 quote
        # 格式
            {\% blockquote [<author>[, <source>]] [<link>] [<source_link_title>] %\}
            <content>
            {\% endblockquote %\}
        # 样例
            - 不加任何形式的引用

                Lorem ipsum dolor sit amet, consectetur adipiscing elit. 
                Pellentesque hendrerit lacus ut purus iaculis feugiat. 
                Sed nec tempor elit, quis aliquam neque. 
                Curabitur sed diam eget dolor fermentum semper at eu lorem.
            {\% endblockquote %\}
        # 样例
            - 注明作者
            {\% blockquote David Levithan, Wide Awake %\}
                Do not just seek happiness for yourself.
                Seek happiness for all.
                Through kindness.
                Through mercy.
            {\% endblockquote %\}
        # 样例
            - 引用 Twitter
            {\% blockquote @DevDocs https://twitter.com/devdocs/status/356095192085962752 %\}
                NEW: DevDocs now comes with syntax highlighting.
                http://devdocs.io
            {\% endblockquote %\}
        # 样例
            - 引用网络上的文章

    3. 代码块 Code Block
        # 在文章中插入代码。
        # 别名 code
        # 格式
            {\% codeblock [<title>] [lang:<language>] [<url>] [<link_text>] [<additional_option>[ ...]] %\}
                <code_snippet>
            {\% endcodeblock %\}


        # <additional_option>
            - 格式
                <option>:<value>
            - 可用选项
                line_number:true            <== 显示行号    
                highlight:true              <== 开启高亮
                first_line:1                <== 指定第一行的行号
                mark                        <== 指定哪些行开启高亮，如 mark:1,4-7,10
                wrap:true                   <== 将代码块包裹在 <table> html 标签中
        # 样例
            - 普通的代码块
            {\% codeblock %\}
                alert('Hello World!');
            {\% endcodeblock %\}
        # 样例
            - 指定语言
            {\% codeblock lang:objc %\}
                [rectangle setX: 10 y: 10 width: 20 height: 20];
            {\% endcodeblock %\}
        # 样例
            - 带有 title
            {\% codeblock Array.map %\}
                array.map(callback[, thisArg])
            {\% endcodeblock %\}
        # 样例
            - 带有 title 和 url
            {\% codeblock _.compact http://underscorejs.org/#compact Underscore.js %\}
                _.compact([0, 1, false, 2, '', 3]);
                => [1, 2, 3]
            {\% endcodeblock %\}

知识点：正则表达式介绍

知识点：正则表达式基础字符

    1. 点  
        # .         ，除了换行符外的任意单个字符  
    2. 字符集合  
        # 介绍  
            - 字符集合中，除 -、^、[ ]、\ 以外，其他所有字符都恢复为普通字符  
            - 字符集合中，只能匹配单个字符，语法上也限定了其只能匹配单个字符  
        # []        ，字符集和，匹配方括号中包含的字符  
        # [^]       ，字符集和补集，匹配除了在方括号中的字符外的其他字符  
        # [a-h]     ，指定字符范围，按照 ascii  
        # [^a-h]    ，指定否定字符范围，按照 ascii  
    3. 重复次数  
        # *     ，匹配重复 0 个到无数个前字符  
        # +     ，匹配重复 1 个到无数个前字符  
        # ?     ，匹配重复 0 个或 1 个前字符  
        # {n}   ，匹配重复 n 个前的字符  
        # {n,}  ，匹配重复 n 个到无数前字符  
        # {n,m} ，匹配重复 n 个到 m 个前字符  
        # ?  
            - 懒惰匹配 lazy，尽可能匹配最短的字符串，在重复次数符号后再加一个问号，如 *?、+?、{n,}?  
            - 默认是贪婪匹配 Greedy，尽可能匹配最长的字符串  
    4. 子字符串  
        # (xyz)     ，匹配与 xyz 完全相等的字符串  
        # (a|b|c)   ，匹配与 a 完全相同的字符，或与 b 完全相同的字符，或与 c 完全相同的字符  
        # (a|b|xyz) ，匹配与 a 完全相同的字符，或与 b 完全相同的字符，或与 xyz 完全相同的字符串  
    5. 或运算  
        # |     ，或运算符，两侧可接正则表达式，匹配符合前正则表达式所指定的字符，或匹配符合后正则表达式所指定的字符  
    6. 锚点  
        # ^     ，行首  
        # $     ，行末  
    7. 转义字符  
        # \

知识点：正则表达式功能字符

    1. 字母和数字  
        # \w    ，匹配所有字母数字加下划线，等同于 [a-zA-Z0-9_]  
        # \W    ，匹配所有非字母数字和下划线，即除了下划线外的所有符号，等同于 [^\w]  
        # \d    ，匹配数字，等同于 [0-9]  
        # \D    ，匹配非数字，等同于 [^\d]  
    2. 空格字符  
        # \b    ，匹配一个单词边界，如 <space>、<period> 等  
        # \B    ，匹配一个非单词边界  
        # \f    ，匹配一个换页符  
        # \r    ，匹配一个回车符 CR  
        # \n    ，匹配一个换行符 LF  
        # \t    ，匹配一个制表符  
        # \v    ，匹配一个垂直制表符  
        # \p    ，匹配 DOS 行终止符 CR/LF，等同于 \r\n  
        # \s    ，匹配所有空格字符，等同于 [ \t\n\f\r\p]  
        # \S    ，匹配所有非空格字符，等同于 [^\s]

知识点：POSIX 方括号表达式 POSIX Bracket Expressions

    1. 使用方法  
        # grep '[[:digit:]]'  
    2. [:alnum:]    ，代表英文字母和数字      ，亦即 A-Z、a-z、0-9  
    3. [:alpha:]    ，代表英文字母             ，亦即 A-Z、a-z  
    4. [:upper:]    ，代表英文大写字母           ，亦即 A-Z  
    5. [:lower:]    ，代表英文小写字母           ，亦即 a-z  
    6. [:digit:]    ，代表数字                   ，亦即 0-9  
    7. [:xdigit:]   ，代表 16 进制会用到的字符 ，亦即 0-9、A-F、a-f  
    8. [:blank:]    ，代表空白                   ，亦即 <space>、<tab>  
    9. [:punct:]    ，代表标点符号             ，亦即 "、'、?、!、;、:、#、$ 等等  
    10. [:print:]   ，代表任何可打印字符  
    11. [:cntrl:]   ，代表控制键              ，亦即 CR、LF、<tab>、<del> 等等  
    12. [:graph:]   ，代表空白以外其他所有按键  
    13. [:space:]   ，代表任何会产生空白的字符   ，包括 <space>、<tab>、CR 等等

知识点：正则表达式捕获组 Capturing Groups 和回溯引用 backreference

    1. 介绍  
        # 捕获组，写在查找字符串中  
        # 回溯引用，写在替换字符串中  
        # 通过使用捕获组和回溯引用，在进行字符串替换时，保留原始字符串中的一部分  
            - 范例  
                原文：aaa123bbb789ccc  
                查找：aaa([1-3]*)bbb([7-9]*)ccc  
                替换：xxx\1yyy\2zzz  
                结果：xxx123yyy789zzz  
        # 定义捕获组时，要注意捕获组前方的重复次数，如 * +，可能因为贪婪匹配而导致捕获组为空，可以改为懒惰匹配  
            - 范例  
                原文：aaa123bbb789ccc  
                查找：.*([1-3]*)bbb([7-9]*)ccc  
                替换：xxx\1yyy\2zzz  
                结果：xxxyyy789zzz  
                查找：.*?([1-3]*)bbb([7-9]*)ccc  
                替换：xxx\2yyy\1zzz  
                结果：xxx123yyy789zzz  
    2. 捕获组格式  
        # (<reguler_expression>)  
            - 定义捕获组  
            - 每个 (<reguler_expression>) 定义一个区域  
    3. 回溯引用格式  
        # \<N>  
            - 定义回溯引用，N 为 1-9 这九个数，指向第 1-9 个捕获组  
            - 指向未定义的捕获组时，结果为空值  
            - 在 perl 和 python 中，N 可以指向第 1-99 个捕获组  
        # ${N}  
            - 定义回溯引用，N 为 1-99，指向第 1-99 个捕获组  
            - 指向未定义的捕获组时，结果为空值  
    4. 命名的捕获组

知识点：正则表达式反向引用 Backreferences

    3. 向后引用 back reference  
        # (<reguler_expression>)\<N>  
            - 该子字符串与正则表达式的第 N 个用括号围起来的捕捉群 (capture group) 子表达式 (subexpression) 匹配  
            - N 是从 1 开始的十进制正整数，其上限可能是 9、31、99 甚至无限  
            - 范例  
                '(.)\1' 表示匹配两个连续的相同字符

知识点–概念：正则表达式断言 lookaround

    1. 断言介绍  
        #  
    1. 正先行断言 positive lookahead  
        # <search_reguler_expression>(?=<lookaround_reguler_expression>)  
            - <search_reguler_expression>，搜索目标字符串，返回值为搜索目标字符串的匹配项  
            - <lookaround_reguler_expression>，断言判断式  
            - 要求目标字符串后面紧跟符合断言判断式的字符串  
            - 范例  
                'g(oo|ol)d(?=(ing|ed))'，搜索 good 或 gold，并且后面均紧接着 ed 或 ing  
    2. 负先行断言 negative lookahead  
        # <search_reguler_expression>(?!<lookaround_reguler_expression>)  
        # 目标字符串后面紧跟不符合断言判断式的字符串  
    3. 正后发断言 positive lookbehind  
        # (?<=<lookaround_reguler_expression>)<search_reguler_expression>  
        # 目标字符串前面为符合断言判断式的字符串  
    4. 负后发断言 negative lookbehind  
        # (?<!<lookaround_reguler_expression>)<search_reguler_expression>  
        # 目标字符串前面为不符合断言判断式的字符串
